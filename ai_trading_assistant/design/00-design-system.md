# 00. 디자인 시스템 (Global Foundation)

**AI Trading Assistant — Global Design Foundation v1.0**
적용 범위: 전체 10개 페이지(Dashboard / Macroeconomics / News / Stock / Financial Statements / Technical Analysis / Asset / Portfolio / Settings 및 하위 화면)의 단일 기준 문서.
기준 해상도: Desktop First, 1920×1080. 다크 테마 단일 운영(라이트 테마 없음).

---

## 0-1. 제품 성격 정의

Bloomberg Terminal의 정보 밀도, TradingView의 데이터 시각화, Apple의 위계와 명료함, Linear의 절제와 정밀함을 결합한 **개인 투자자용 프리미엄 트레이딩 워크스페이스**.

- 사용자는 하루 수 회, 짧게는 30초(장전 브리핑 확인), 길게는 30분(종목 분석) 단위로 접속한다.
- 따라서 모든 화면은 "3초 안에 핵심 파악 → 필요 시 파고들기(Drill-down)"의 2단 구조를 갖는다.
- 화면은 정보가 많아 보여야 하지만, 절대 복잡해 보여서는 안 된다. 이 모순의 해법은 **색의 절제 + 타이포 위계 + 균일한 그리드**다.

## 1. 디자인 원칙 (Design Principles)

### P1. 숫자가 주인공이다 (Numbers First)
모든 화면에서 가장 큰 시각 요소는 장식이나 일러스트가 아니라 **숫자**여야 한다.
- 실제 작동 예: Dashboard의 KOSPI 카드에서 `2,674.31`은 32px Bold, 라벨 "KOSPI"는 12px Gray다. 라벨이 숫자보다 크거나 진한 조합은 금지.
- 실제 작동 예: 카드 배경에 그라데이션·일러스트를 넣자는 요구가 오면 이 원칙으로 기각한다.

### P2. 색은 의미다 (Color = Meaning)
색은 오직 4가지 목적에만 쓴다 — ① 등락 방향 ② 상태(경고/오류/신선도) ③ 핵심 CTA ④ 차트 시리즈 구분. 장식 목적의 색 사용은 전면 금지.
- 실제 작동 예: 뉴스 카드의 카테고리 태그는 색이 아니라 회색 아웃라인 배지로 구분한다. 색을 쓰는 순간 "등락인가?"라는 오독이 생기기 때문이다.
- 규칙 문장(§2-6)으로 강제한다.

### P3. 신선도는 데이터의 일부다 (Freshness is Data)
값과 시각(as-of)은 한 몸이다. 언제 데이터인지 모르는 숫자는 틀린 숫자보다 위험하다.
- 실제 작동 예: 야간선물 값이 시장 마감 후에도 흰색 큰 숫자로 표시되어 "지금 값"처럼 오독된 실제 사고가 있었다. 이후 모든 시세 컴포넌트는 세션 인식(Session-aware) 렌더링을 의무화한다(§9-4).

### P4. 위계는 밝기로, 구분은 보더로 (Hierarchy by Luminance)
다크 테마에서 그림자는 거의 보이지 않는다. 깊이는 **표면 밝기 단계 + 1px 보더**로 표현하고, 그림자는 오버레이 레이어(드롭다운·모달)에만 허용한다(§4-3).
- 실제 작동 예: 카드 위에 뜨는 팝오버는 카드보다 한 단계 밝은 표면(#2A3648)과 그림자를 함께 갖는다. 같은 평면의 카드끼리는 그림자를 절대 쓰지 않는다.

### P5. 3클릭 도달, 1화면 요약 (3-Click Reach)
어떤 정보든 3클릭 이내 도달해야 하며, 각 페이지 첫 화면(스크롤 0 상태)은 그 페이지의 결론 요약을 담는다.
- 실제 작동 예: "삼성전자 부채비율" 도달 경로 = 헤더 검색에서 종목 선택(1클릭, Enter) → 선택 즉시 Stock Hub 패널 오픈(추가 클릭 없는 인터스티셜) → 패널 내 "재무제표 분석" 버튼(2클릭). 3클릭 예산 중 2클릭만으로 도달하는 여유 사례이며, 이보다 깊어지는 IA는 재설계 대상. (※ "Financial Statements 탭"과 같이 실재하지 않는 컴포넌트를 예시에 인용하지 않는다 — 05 Stock Hub §1-1 기준 실제 IA를 따른다.)
- 실제 작동 예: Macroeconomics 첫 화면 상단엔 항상 "오늘의 매크로 한 줄 요약" Hero 카드가 온다.

#### 2-3-1. `accent.violet` "해석 레이어" 사용 원칙 (특별함 유지 규칙)

`accent.violet`은 "시스템이 데이터를 대신 해석해 짚어준 판단"이라는 단 하나의 신호를 위해 존재하는 색이다. 좌측 3px 바 + overline + 문장으로 구성된 동일한 박스가 페이지마다 반복 등장하면, 그 자체가 하나의 정형화된 위젯으로 소비되어 신호력이 사라진다. 다음 원칙으로 특별함을 지킨다.

- **전용 페이지 우선 배정**: 좌측 바를 갖춘 "정식" 해석 레이어 박스는 **Dashboard(모닝 요약)와 Financial Statements(진단 요약)** 두 곳을 1차 용도로 삼는다. 이 두 페이지가 제품에서 가장 응축된 판단이 필요한 지점이기 때문이다.
- **그 외 페이지는 기본을 일반 캡션으로**: Macroeconomics, News, Asset 등 나머지 페이지에서 유사한 시스템 코멘트가 필요하면 기본값은 좌측 바 없는 **일반 캡션**(`type.caption` + `text.secondary`)이다. 색 예산이 남았다는 이유만으로 violet 박스를 기본 채택하지 않는다.
- **불가피하게 확장할 경우 형태를 변주한다**: 정말 강한 강조가 필요한 예외 페이지에서는 좌측 바 대신 상단 보더, 배경 틴트 유무, 아이콘 유무·폭 등 최소 1가지 이상의 형태 요소를 다르게 하여 "동일 박스의 복제"로 보이지 않게 한다. **좌측 3px 바 + 회색 텍스트의 완전히 동일한 조합을 5개 이상 페이지에 반복 적용하는 것을 금지한다.**
- 색 자체(#A78BFA)의 용도 제한("시세·등락 관련 요소 금지")은 위 §2-3 표 규칙을 그대로 따른다.

### 2-5. 등락(Market) 토큰 — ★기본값 결정

**결정: 기본값은 한국 관습 — 상승 = 빨강, 하락 = 파랑.** Settings에서 "글로벌 모드(상승=초록/하락=빨강)"로 전환 가능.

근거:
1. **주 사용자의 지배적 멘탈 모델**: 본 제품의 1차 사용자는 국내 HTS/MTS(키움 등)로 훈련된 한국 개인 투자자다. 등락 색 오독은 이 제품에서 발생 가능한 가장 비싼 실수이며, 기본값은 사용자의 기존 반사신경을 따라야 한다.
2. **브랜드 색과의 충돌 회피**: Primary Green(#10B981)은 CTA·브랜드 색이다. 글로벌 모드를 기본으로 하면 "초록 = 상승"과 "초록 = 버튼"이 한 화면에서 충돌한다. 한국 모드에서는 등락(빨강/파랑)과 브랜드(초록)가 완전히 분리되어 색 체계가 더 깨끗하다.
3. **한·미 통합 화면의 일관성**: 미국 종목도 한국 관습 색으로 통일 표기한다. 시장별로 색 관습을 섞는 것이 최악의 선택이다.

| 토큰명 | 한국 모드(기본) | 글로벌 모드 | 용도 |
|---|---|---|---|
| `market.up` | #F87171 | #34D399 | 상승 텍스트·아이콘 (▲와 항상 병행) |
| `market.up-bg` | #F87171 12% 오버레이 | #34D399 12% | 상승 셀·배지 배경 |
| `market.down` | #60A5FA | #F87171 | 하락 텍스트·아이콘 (▼와 항상 병행) |
| `market.down-bg` | #60A5FA 12% 오버레이 | #F87171 12% | 하락 셀·배지 배경 |
| `market.flat` | #9CA3AF | #9CA3AF | 보합 (0.00%) |

- 등락 색은 텍스트·아이콘·미니 배경에만 쓴다. 카드 전체 배경을 등락 색으로 칠하는 것 금지.
- 글로벌 모드 전환 시 차트 캔들 색도 동일 규칙으로 일괄 전환된다.

### 2-6. 색 절제 원칙 (규칙 문장)

> **R1. 한 화면(뷰포트 기준)에 동시에 존재하는 '의미 색'은 4종 이내로 제한한다.** (등락 2색 + 상태 1색 + CTA 1색이 상한)
> **R2. 장식 목적의 유채색 사용을 금지한다.** 구분이 필요하면 회색 단계·보더·타이포 굵기를 먼저 쓴다.
> **R3. 핵심 CTA(Primary Green 채움 버튼)는 한 화면에 1개만 존재한다.** 나머지 액션은 아웃라인/고스트 버튼.
> **R4. 등락 색은 반드시 ▲▼ 기호 또는 +/− 부호와 병행한다.** 색 단독으로 방향을 전달하지 않는다.
> **R5. 차트 시리즈 색은 §2-7에 정의된 전용 5색 팔레트(`chart.series-1~5`)만 사용한다.** 등락(`market.*`)·상태(`status.*`)·브랜드·액센트(`brand.*`/`accent.*`) 토큰과 헥스값이 절대 겹치지 않는 별도 계열이며, 6번째 시리즈부터는 팔레트의 회색 항목(`chart.series-5`) 재사용 + 라벨 직접 표기.
> **R6. 색 토큰을 신설·변경할 때는 반드시 §2-1~§2-8의 기존 전체 헥스값과 대조해 중복 여부를 검증한다(빌드 타임 린트 대상).** 하나의 헥스값이 등락·상태·브랜드·차트라는 서로 다른 의미 카테고리에 동시에 배정되는 것은 그 자체로 결함이다 — 색은 의미다(P2)라는 원칙이 색상표 레벨에서 깨지는 것을 사전 차단한다.

### 2-7. 차트 시리즈 토큰 (Chart Series Palette)

차트 시리즈 색은 "무엇이 좋고 나쁜지"를 말하지 않는 순수 구분용 색이어야 한다. 등락·상태·브랜드·액센트 의미 토큰과 헥스값이 하나도 겹치지 않는 전용 5색 팔레트를 별도로 정의한다(R5/R6 근거 — 구 팔레트는 `market.down`(한국 모드 #60A5FA)·`status.info`(#60A5FA)·`status.success`/`brand.primary-hover`(#34D399)·`badge.live`(#34D399)와 헥스가 겹쳐 "하락"과 "차트 시리즈"가 동일 색으로 오독될 수 있었다).

| 토큰명 | HEX | 색상 계열 | 용도 |
|---|---|---|---|
| `chart.series-1` | #2DD4BF | 청록(Teal) | 1번째 시리즈(기본) — 예: 90일 추이 메인 라인 |
| `chart.series-2` | #E879F9 | 자주(Fuchsia) | 2번째 시리즈 |
| `chart.series-3` | #FB923C | 주황(Orange) | 3번째 시리즈 |
| `chart.series-4` | #818CF8 | 남보라(Indigo) | 4번째 시리즈 — `accent.violet`(#A78BFA)과 계열은 유사하나 헥스·색조 모두 구분되며, 해석 레이어 전용색과 혼동되지 않도록 채도·명도를 달리함 |

- 등락 의미가 있는 단일 계열 차트(예: 종목 하나의 가격 라인만 그릴 때)는 이 팔레트가 아니라 `market.up`/`market.down`/`text.primary`를 상황에 맞게 쓴다 — 차트 팔레트는 **복수 시리즈를 서로 구분**할 때만 사용한다.

```
└─────────────────────────────────────────┘
```

- 헤더 액션: 아이콘 버튼 28×28px(히트 영역 32px), 최대 2개(새로고침·더보기). 3개 이상 필요하면 "더보기" 메뉴로 수납.
- 타임스탬프는 모든 데이터 카드에서 **생략 불가**(§9-2).
- 카드 보더 `border.default` 1px, 배경 `surface.card`, radius 12px. 카드 전체가 클릭 대상일 때만 호버 배경 적용.

### 8-2. 카드 등급 체계

| 등급 | 그리드 스팬 | 최소 높이 | 패딩 | 대표 타이포 | 용도 예시 |
|---|---|---|---|---|---|
| **Hero** | 8~12col | 320px | 24px | `num-display` 32px | 모닝리포트 요약, 지수 종합 현황 |
| **Standard** | 4~6col | 220px | 20px | `num-lg` 24px | 환율·유가 카드, 뉴스 리스트 카드, 차트 카드 |
| **Compact** | 3~4col | 120px | 16px | `num-md` 16px + 스파크라인 | 단일 지표(KOSPI, S&P 500, 달러인덱스) |
| **Row** | 부모 카드 내부 | 행 52px (밀집 40px) | 좌우 16px | `num-md` 16px | 관심종목 리스트, 뉴스 행, 매매일지 행 |

- **등급 간 타이포 위계 규칙**: Compact는 Standard보다 한 단계 낮은 `num-md`(16px)를 대표 타이포로 쓴다. 그리드 스팬·최소 높이·패딩만으로는 두 등급이 나란히 배치됐을 때 어느 쪽이 더 비중 있는 정보인지 구분되지 않으므로, 숫자 크기 자체가 등급 신호의 1차 채널이 되도록 강제한다. Hero(`num-display` 32px) → Standard(`num-lg` 24px) → Compact/Row(`num-md` 16px)로 이어지는 3단 위계를 어떤 조합에서도 깨지 않는다.

### 8-3. Compact 카드 와이어프레임 (예시 데이터)

```
┌────────────────────────────┐
│ S&P 500        07/09 마감 기준│  ← 라벨 12px + 신선도 캡션
│ 6,214.85                    │  ← num-md 16px, text.primary
│ ▲ +32.14 (+0.52%)  ∿∿∿∿╱   │  ← 등락 13px market.up + 스파크라인 96×28px

```

### 8-4. Row 예시 (관심종목)

```
│ 삼성전자  005930   ₩84,300   ▲ +0.95%   [스파크라인]  │ 52px
│ NVIDIA   NVDA     $192.42   ▼ −1.12%   [스파크라인]  │
```
- 열 정렬: 종목명 좌측, 숫자 열은 전부 우측 정렬(tabular). 행 호버 시 `surface.card-hover`, 클릭 시 Stock 상세로 이동.

### 8-5. 전역 공유 컴포넌트 계약 — 계좌/자산 배분 스택 바

"계좌별 자산 배분"(자산별 비중) 스택형 바는 Asset·Portfolio 등 복수 페이지가 동일한 정보(비중 breakdown)를 표현하는 **단일 전역 컴포넌트**다. 페이지별 색 예산 소진 여부 같은 우연한 사정으로 한쪽은 무채색, 다른 쪽은 유채색으로 그리는 것을 금지하고, 다음 계약을 모든 페이지가 예외 없이 상속한다.

| 항목 | 규칙 |
|---|---|
| 색 | 세그먼트 색은 `chart.series-1~5`(§2-7) 순서로 고정 배정한다. 항목이 6개 이상이면 6번째부터 `chart.series-5`(회색)를 재사용하고 라벨로 구분한다 |
| 라벨 | 세그먼트 폭이 충분하면 바 내부에, 좁으면 바 하단 범례에 계좌명·비중(%)을 **항상** 텍스트로 병기한다 — 색만으로 구분하지 않는다(§11-2와 동일 원칙) |
| 정렬 순서 | 비중 큰 순 → 작은 순으로 좌→우 배치, 동률 시 가나다순 |
| 높이 | 24px(단독 위젯) / 16px(카드 내부 축소형) |
| 호버 | 세그먼트 호버 시 Level 2 툴팁으로 정확한 금액·비중 표시 |
| 예외 금지 | 페이지별로 무채색/유채색을 임의로 다르게 적용하지 않는다. 색 예산(§2-6 R1) 소진 등 개별 페이지 사정으로 동일 컴포넌트의 색 처리를 바꾸지 않는다 |

이 컴포넌트는 Asset·Portfolio 문서가 공통으로 인용하는 전역 정의이며, 두 문서 모두 본 계약을 그대로 상속한다(개별 변형 금지). §2-7의 팔레트 재정의로 등락·상태 색과의 헥스 충돌이 해소되었으므로, 유채색 팔레트를 계좌/자산 배분처럼 순수 비교·구분 목적에 안전하게 재사용할 수 있다(P2의 ④ 차트 시리즈 구분 용도에 해당).

## 9. 상태 시스템 (States & Data Freshness)

### 9-1. 로딩 스켈레톤

- 스켈레톤은 최종 레이아웃과 동일한 골격(제목 바, 숫자 블록, 스파크라인 자리)을 유지한다 — 로드 후 요소 점프 금지.
- 블록: `skeleton.base` 배경, radius 4px(텍스트 라인 높이 12px) / 6px(차트 영역). 시머는 `skeleton.highlight`가 좌→우 1400ms 주기.
- 1.5초 초과 시 카드 헤더에 "불러오는 중" 캡션 추가. 스피너는 버튼 내부 로딩에만 사용.
- **잠금 콘텐츠 게이트(Asset·Portfolio 본문)**: 일반 로딩 스켈레톤과 달리, 배경에 실제 레이아웃 스켈레톤을 깔고 그 위에 `overlay.gate`(#000000 40%, §2-8)를 씌운다 — 모달 전용 `overlay.scrim`(60%)을 재사용하지 않는다. 60% 불투명 검정 위에서는 스켈레톤 블록의 형태가 거의 지각되지 않아 "점프 없는 해제"라는 목적을 달성하지 못하기 때문이다. 배경 스켈레톤 없이 순수 `overlay.gate`와 잠금 아이콘·CTA만 두는 구성도 허용되며, 이 경우 해제 시 콘텐츠는 페이드-인 + 8px 상승(§10-2 오버레이 등장 규칙)만으로 점프 없는 전환을 구현한다.

### 9-2. 데이터 신선도 (★ 핵심 규칙)

> **N1. 타임스탬프 없는 시세·지표 숫자는 렌더링 금지.** 모든 데이터 카드는 as-of 시각(`07/10 07:12 KST` 형식)을 헤더 또는 값 옆에 반드시 표기한다.
> **N2. 신선도는 4단계로 자동 판정하고 시각적으로 구분한다.**

| 상태 | 판정 기준(데이터별 기대 갱신 주기 T 대비) | 시각 표현 |
|---|---|---|
| **FRESH** | 경과 < T | 값 `text.primary`, 배지 없음 (세션 중이면 라이브 점 허용) |
| **DELAYED** | T ≤ 경과 < 3T | "지연" 배지(`accent.yellow` 아웃라인, 12px), 값은 흰색 유지 |
| **STALE** | 경과 ≥ 3T | **값 전체를 `text.secondary`로 강등** + "오래된 데이터" 배지(`accent.yellow` 채움 12%), 툴팁에 마지막 갱신 시각 |
| **CLOSED-SNAPSHOT** | 해당 시장·세션 마감 후 | 값 `text.primary` 유지하되 "07/09 마감 기준" 라벨 필수, 라이브 점·펄스 전면 금지 |

### 9-3. 빈 상태 · 에러 상태

- **빈 상태**: 카드 중앙에 아이콘 32px `text.tertiary` + 메시지 1줄(`type.body`, `text.secondary`) + 필요 시 액션 링크 1개. 예: "오늘 작성된 매매일지가 없습니다 — 새 일지 작성".
- **에러 상태**: `status.danger` 아이콘 20px + 원인 메시지 + "다시 시도" 고스트 버튼. **캐시된 직전 데이터가 있으면 빈 화면 대신 STALE 스타일로 이전 값을 유지 표시**하고 에러는 배지로 알린다(정보 제로 상태 회피 원칙).

### 9-4. 세션 인식 렌더링 (★ 야간선물 사고 방지 규칙)

배경: 야간선물(CME 연계 등) 값이 시장 마감 후에도 흰색 큰 숫자로 남아 "현재 실시간 값"처럼 오독된 실제 사고가 있었다. 이를 컴포넌트 계약으로 구조적으로 차단한다.

> **S1. 모든 시세 데이터 소스는 '유효 세션' 정의를 갖는다.** (예: 야간선물 18:00–다음날 05:00 KST, 미국 정규장 09:30–16:00 ET)
> **S2. 현재 시각이 유효 세션 밖이면 '라이브처럼 보이는' 스타일을 금지한다** — 라이브 점, 펄스, 흰색 대형 숫자 단독 표시 모두 불가. 값은 CLOSED-SNAPSHOT 형식("07/10 05:00 마감 기준")으로만 표시한다.
> **S3. 등락률은 반드시 기준가 기준을 병기한다.** 예: "▲ +0.42% (전일 정규장 종가 대비)". 기준가를 알 수 없으면 등락률 자체를 숨기고 절대값만 표시한다.
> **S4. 세션 경계(개장 직전·마감 직후)에는 상태 전환을 값 갱신보다 우선한다** — 새 데이터가 오기 전이라도 마감 시각이 지나면 즉시 CLOSED-SNAPSHOT 스타일로 전환한다.

### 10-2. 규칙

- 오버레이 등장: 불투명도 0→1 + 아래에서 8px 상승. 퇴장은 120ms로 더 빠르게.
- 값 변경 플래시: 시세 갱신 시 해당 셀 배경에 `market.*` 12% 틴트를 300ms 1회만. **연속 점멸 금지.**
- 차트 등장 애니메이션은 300ms 이내 1회. 스크롤 연동 애니메이션·패럴랙스·바운스 전면 금지.
- 무한 반복 애니메이션은 라이브 점 펄스(2s 주기, 세션 중 한정)와 스켈레톤 시머만 허용.
- OS의 "동작 줄이기" 설정 시 모든 모션 제거(즉시 전환), 시머는 정적 톤 변화로 대체.

### 10-3. 호버·포커스

- 호버: 배경 변화(`surface.card-hover`)가 기본. 스케일 변형·그림자 추가 금지.
- 포커스: 모든 인터랙티브 요소에 `focus.ring`(2px #34D399, offset 2px) 일관 적용. 마우스 클릭 포커스에는 링 생략, 키보드 포커스에만 표시.

## 11. 접근성 (Accessibility)

### 11-1. 대비율 기준

- 본문 텍스트: WCAG AA **4.5:1 이상**, 대형 텍스트(18px+ 또는 14px Bold+)·UI 요소: **3:1 이상**.
- 검증된 조합: `text.primary`(#F9FAFB) on `surface.card` ≈ 13:1, `text.secondary`(#9CA3AF) on card ≈ 5.3:1, `text.tertiary`(#6B7280) on card ≈ 3.2:1 → **tertiary는 12px 이상 비필수 정보 전용**.
- 등락 색(#F87171 / #60A5FA)은 카드 배경 대비 약 4:1 수준 → 등락 값은 항상 **13px 이상 + 500 굵기 이상 + ▲▼ 기호 병행**으로 보완한다.

### 11-2. 색맹 대응

- 등락은 색 단독 전달 금지(R4). ▲▼ 기호와 +/− 부호가 1차 정보, 색은 보조 채널.
- 한국 모드의 빨강/파랑 쌍은 적록색맹에게 글로벌 모드(초록/빨강)보다 유리하다 — 기본값 결정(§2-5)의 부수 이점.
- 차트에서 시리즈 구분은 색 + 라벨 직접 표기(범례 의존 최소화), 필요 시 선 스타일(실선/점선) 병행.
- 상태 배지는 색 + 텍스트 라벨("지연", "마감 기준") 병행 — 아이콘·색 단독 배지 금지.

### 11-3. 키보드 내비게이션

| 키 | 동작 |
|---|---|
| `Ctrl/⌘ + K` | 글로벌 검색(커맨드 팔레트) |
| `Tab / Shift+Tab` | 논리적 순서(사이드바 → 헤더 → 콘텐츠 좌상→우하) 포커스 이동 |
| `Esc` | 오버레이 닫기(모달 → 팝오버 → 검색 순 역순 해제) |
| `[` | 사이드바 접기/펼치기 |
| `↑↓ + Enter` | 리스트·검색 결과 탐색 및 진입 |

- 첫 Tab에서 "본문으로 건너뛰기" 스킵 링크 제공.
- 모달 열림 시 포커스 트랩, 닫힘 시 트리거 요소로 포커스 복귀.
- 최소 히트 영역 32×32px. 스크린리더용으로 등락 값은 "플러스/마이너스"를 읽을 수 있는 레이블 텍스트를 제공한다(기호만 노출 금지).

## 12. 개선 포인트 (구현 제약 관련 — 요약)

- 서비스는 정적 GitHub Pages 기반(서버·DB 없음)이므로, §9의 신선도 판정은 **빌드 시점에 기록된 as-of 타임스탬프 + 열람 시점의 클라이언트 시계 비교**로 수행하는 것을 전제로 설계했다. 데이터 파일에는 값과 함께 생성 시각·유효 세션 메타가 항상 포함되어야 한다.
- Asset·Portfolio의 비밀번호 게이트는 정적 환경 특성상 열람 차단(프라이버시 가림) 수준임을 Settings 내 고지 문구로 명시한다.
- 갱신 주기가 배치(모닝리포트 1일 1회, 뉴스 수 시간 간격)인 데이터는 FRESH 기준 주기 T를 소스별로 다르게 설정한다 — 실시간 소스와 동일 잣대 금지.
