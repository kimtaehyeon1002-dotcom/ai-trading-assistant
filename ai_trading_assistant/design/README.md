# AI Trading Assistant — UX/UI 리디자인 설계 문서

개인 투자자를 위한 한국+미국 통합 프리미엄 투자 플랫폼의 웹디자인 설계 문서.
**디자인/UX 설계만 담으며, 구현 코드(HTML/CSS/JS 등)는 포함하지 않는다.**

- 참고 철학: Bloomberg Terminal(정보 밀도) · TradingView(데이터 시각화) · Apple(위계) · Linear(절제)
- 기준: Desktop First 1920px, 다크 테마 단일 운영
- 이 폴더는 GitHub Pages 배포 대상(`docs/`)이 아니다 — 순수 설계 문서 보관용.
- **상태: v1.1 (검증 완료)** — 3개 렌즈(토큰/명칭 일관성, 3클릭·동선, 프리미엄 완성도)로 11개 문서 전체를 교차 검증해 38건(critical 8 · major 22 · minor 8)을 찾아 전량 반영했다. 상세 내역은 하단 [검증 리포트](#검증-리포트-v11) 참조.

## 구현 설계 문서 (20번대 — 코드 없음, 설계만)

디자인(00~10)을 현행 코드베이스(정적 GitHub Pages + Python build + Jinja2 + 바닐라 JS)에 얹기 위한 단계별 계획. 3개 문서 모두 현행 코드 실측 → 격차 분석 → 3관점 아키텍처 심사 → 2렌즈 검증(23건 지적 반영)을 거친 최종본이며, **계좌 4종 체계**(키움 단타·스윙 / 한투 위탁 미국주식 / 한투 ISA ETF / BYBIT 암호화폐, KB증권 삭제)를 정본으로 한다.

핵심 정본 결정 (3문서 공통):
- **자산 평문 스냅샷은 커밋 금지** — `data/snapshots/` .gitignore 등재 + pre-commit/CI 이중 가드. 발행물은 암호문(`assets.enc.json`) 단독.
- **신선도 = 4상태 정본** (FRESH ≤2T / DELAYED / STALE / CLOSED-SNAPSHOT) — 00 디자인 시스템 기준으로 통일, 미정의 시간 구간 소거.
- **자산 수집 전량 자동화** — KB 삭제로 수기 입력 소스 클래스 소멸, T=24h 단일 신선도 규칙.
- **기존 URL·아카이브 전수 보존** — base.html 동결, 아카이브 CSS는 동결 자산으로 영구 배포.

## 핵심 결정 요약

- **등락 색 기본값 = 한국 관습(상승 빨강 ▲ / 하락 파랑 ▼)**, Settings에서 글로벌 모드 전환. Primary Green(#10B981)은 CTA/브랜드 전용으로 등락과 역할 분리.
- **색 절제 규칙**: 한 화면 의미색 4종 이내, Primary 채움 버튼은 화면당 1개, 등락 색은 반드시 ▲▼/+− 기호 병행.
- **데이터 신선도**: 모든 시세 컴포넌트는 세션 인식(장전/장중/장후) 렌더링 의무 — as-of 타임스탬프 + FRESH/CLOSED-SNAPSHOT/STALE 상태. 야간선물 stale 값 오독 사고의 구조적 방지.
- **그리드**: 사이드바 240px(접힘 64px) + 헤더 64px + 콘텐츠 최대 1600px, 12컬럼/거터 24px.
- **3클릭 규칙**: 어떤 정보든 3클릭 이내. 예) 종목 재무 = 헤더 검색(1클릭) → Stock Hub 패널 자동 오픈(0클릭) → 재무제표 분석 버튼(2클릭).

## 검증 리포트 (v1.1)

11개 문서(디자인 시스템 1 + 페이지 10) 전체를 3개 독립 렌즈로 교차 검증했다. 각 렌즈는 서로의 지적 내용을 모른 채 문서 전체를 다시 읽고 판단했다.

| 렌즈 | 검증 대상 |
|---|---|
| 일관성 | 토큰·수치·명칭·패턴이 00 문서 기준과 페이지 간에 정확히 일치하는가 |
| UX·정보 위계 | 3클릭 규칙, 핵심정보 우선순위, 크로스 링크, 게이트 마찰이 실제로 작동하는가 |
| 프리미엄 완성도 | 색 남용, 카드 위계 붕괴, 여백·타이포 문제로 "복잡해 보일" 위험이 있는가 |

**결과: 38건 지적(critical 8 · major 22 · minor 8), 전량 반영 완료.**

### 발견된 대표 문제와 조치

- **Stock Hub 표면 토큰 충돌** — 마스터 스펙(05)은 `surface.card`, 호출부(04·09)는 `surface.elevated`를 각각 명시해 정면 충돌. → 제3의 전용 토큰 `surface.overlay-panel`(#253040)을 신설해 카드<패널<호버<팝오버 4단 밝기 위계로 해소.
- **뉴스 종목 칩의 동작 불일치** — News(03)는 "Stock 페이지 이동", Stock Hub(05)는 "오버레이 패널 오픈"으로 서로 다르게 정의. → News를 05 기준(오버레이 오픈, 기사 컨텍스트 유지)으로 통일.
- **Stock Hub 브레드크럼 규칙 상충** — 05는 "브레드크럼 미변경", 04는 "갱신"으로 정반대 서술. → 05(패널은 페이지 이동이 아니므로 컨텍스트 유지)로 통일.
- **Stock Hub CTA 불변 계약 위반** — 05는 CTA 2개(TradingView/재무제표 분석)로 고정하고 페이지별 변형을 금지했으나, 04는 "기술적 분석" 버튼을 추가해 Coming Soon 상태인 TA(07) 페이지로 유도. → 05 기준 2버튼으로 정정, TA가 정식 출시되기 전까지 해당 CTA 비노출 조건 명문화.
- **차트 팔레트가 의미 색과 충돌** — 기존 5색 차트 팔레트의 색상값이 하락(`market.down`)·정보(`status.info`)·성공(`status.success`) 토큰과 그대로 겹쳐, "파란 선이 하락으로 오독"될 위험. → 등락·상태·브랜드 토큰과 헥스가 전혀 겹치지 않는 `chart.series-1~5` 신규 팔레트로 전면 교체하고, 재발 방지용 검증 규칙(R6) 신설.
- **Dashboard 기본 화면 카드 수 불일치** — 01은 "정확히 5개 카드"로 정의했지만 05의 예시 와이어프레임엔 관심종목·개별 지수 카드가 추가로 그려져 있었음. → 01에 "문서 간 일관성 계약"을 명문화해 다른 문서가 이를 어길 수 없도록 고정.
- **일정 항목의 죽은 링크** — Stock Hub·Portfolio의 "관련 일정" 행이 실제로는 존재하지 않는 "공시 원문/IR 페이지"로 이동한다고 정의되어 있었음. → 유형별(실적=내부 이동, 공시/IR/SEC=외부 새 탭, 배당=비활성) 전역 계약 표로 목적지를 전량 확정.
- **accent.violet 남용** — "해석 레이어" 색이 5개 이상 페이지에서 반복돼 신호력이 옅어짐. → Dashboard·Financial Statements 두 곳으로만 용도를 한정하고 나머지 페이지는 일반 캡션/형태 변주로 대체.
