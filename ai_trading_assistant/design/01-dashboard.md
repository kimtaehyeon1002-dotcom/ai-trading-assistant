# 01. Dashboard

**AI Trading Assistant — Dashboard 페이지 설계 v1.1**
기준 문서: 00. 디자인 시스템(Global Foundation v1.0). 본 문서는 00의 토큰명·수치를 변경 없이 인용한다.
기준 해상도: 1920×1080, Desktop First, 다크 테마 단일.

> **개정 이력 v1.1(리뷰 반영)**: (1)(2) Dashboard 기본 화면이 정확히 5개 카드(Hero/한국지수/미국지수/핵심뉴스/일정)뿐이며 관심종목 카드는 opt-in이라는 계약을 §3-1·§7-2에 명문화하고 다른 문서(05 Stock Hub 등)가 지켜야 할 규칙으로 못박음. (3) ④ 핵심 뉴스 카드의 필터 taxonomy를 News 페이지(03)와 동일하게(전체/미국시장/한국시장/거시경제/속보) 통일하고 'News 전체 보기 →' 이동 시 탭 유지 매핑 규칙을 명시. (4) Dashboard의 뉴스 행 클릭(Hero 열3·④ 카드)이 헤더 글로벌 검색(00 §7-2)과 동일하게 'News 페이지에 기사가 펼쳐진 상태로 도착'하도록 딥링크 계약을 통일. (5) Hero 한 줄 요약과 대표지수의 타이포 위계를 재조정(요약 `type.h2` 20/700 격상, 대표지수 `type.num-lg` 24/600으로 하향)해 "결론이 1순위"라는 설계 의도와 시각 위계를 일치. (6) 타이포 격차 축소를 보완하기 위해 Hero 전용 시각 요소(헤더 톤차·하단 언더라인)를 추가. (8) violet 해석 레이어의 전사 사용 범위를 Dashboard·FS 두 페이지로 한정하는 규칙(00 §2-3-1에 정의)을 이 문서 §3-2에서 정확히 인용하도록 교차 참조를 바로잡음 — 이 페이지가 그 규칙이 실제로 적용되는 두 전용 페이지 중 하나임. 상세 근거는 각 절 본문 참조.

## 1. 전체 레이아웃 설명

### 1-1. 페이지의 임무

Dashboard는 "아침 출근 전 3분 안에 오늘 시장을 파악"하는 화면이다. 정보 설계의 결론은 다음 한 문장으로 요약된다.

> **스크롤 0 상태에서 Morning Report(결론) → 지수(숫자) → 뉴스(맥락) → 일정(다음 행동)의 4단 위계가 한 화면에 전부 들어온다.**

이를 위해 페이지 전체 수직 예산을 1080px 뷰포트에 맞춰 역산 설계했다.

| 구간 | 높이 |
|---|---|
| 헤더(§7) | 64px |
| 페이지 상단 여백 | 24px |
| ① Morning Report Hero | 340px |
| 거터 | 24px |
| ② 한국 지수 + ③ 미국 지수 | 240px |
| 거터 | 24px |
| ④ 핵심 뉴스 + ⑤ 오늘 일정 | 348px |
| **합계** | **1064px ≤ 1080px** |

즉 1080px 모니터에서 **세로 스크롤 없이 5개 카드 전체가 노출**된다. P5(1화면 요약)의 가장 강한 형태다. 이 5카드 구성은 Dashboard 기본 화면의 유일한 정의다 — 관심종목 카드를 포함한 그 외 어떤 카드도 이 화면에 기본 노출되지 않는다(상세 계약은 §3-1 표 하단 및 §7-2 참조. 다른 문서가 Dashboard를 예시로 인용할 때도 이 5카드 구성을 그대로 따라야 한다).

### 1-2. 골격

- 좌측 사이드바 240px 고정, 상단 헤더 64px 고정 — §5-1 그대로. 콘텐츠 영역은 최대폭 1600px, 12컬럼 / 거터 24px / 좌우 마진 32px, 중앙 정렬.
- 컬럼 실폭 약 112px 기준 스팬별 실폭: **12col = 1600px / 8col = 1064px / 6col = 792px / 4col = 520px / 3col = 384px.**
- 행 구성: 1행 Hero 12col 단독 → 2행 6col + 6col → 3행 8col + 4col. 좌상→우하로 갈수록 정보의 시급성이 낮아지는 F-패턴 배열.

### 1-3. 이 페이지의 사이드바·헤더 특이사항 (재정의 아님)

- **사이드바**: `Dashboard` 항목 활성 — §6-2 활성 스펙(좌측 3×20px `brand.primary` 인디케이터 바, 아이콘 `brand.primary`, 라벨 `text.primary` 600) 그대로 적용.
- **헤더 컨텍스트(§7-1)**: 타이틀 "Dashboard" 단독, 브레드크럼 없음(최상위 페이지).
- **콘텐츠 내 `type.h1` 페이지 타이틀은 의도적으로 생략**한다. 헤더 컨텍스트가 타이틀 역할을 대신하며, 절약한 56px(타이틀+여백)이 "무스크롤 한 화면" 예산을 성립시킨다. §3-2의 "h1 페이지당 1개"는 상한이지 의무가 아니다.
- **듀얼 시계·시장 상태(§7-3)**는 이 페이지와 가장 강하게 상호작용한다 — 콘텐츠의 모든 세션 인식 렌더링(§9-4)은 헤더의 세션 상태 점과 동일한 판정을 공유해야 한다(헤더는 "장중"인데 카드는 "마감 기준"인 모순 금지).

### 1-4. 축소 대응(§5-2 준수)

- **1440px**: 거터 20px, 마진 24px. Hero 내부 3열 유지(열폭 축소). 2·3행 스팬 유지.
- **1280px**: 사이드바 자동 접힘(64px), 거터 16px. 스택 순서(위→아래): ① Hero → ② 한국 지수 → ③ 미국 지수 → ⑤ 오늘 일정 → ④ 핵심 뉴스. 일정은 "다음 행동" 정보라 뉴스보다 위로 올린다(보조 정보부터 아래로 미는 §5-2 원칙의 적용).

- **오늘의 핵심 이벤트 1건**(예: 금통위)은 색이 아니라 **굵기 600 + ★ 프리픽스 + 행 상하 1px `border.default`** 로 강조 — 유채색 예산을 쓰지 않는 강조(R2).
- 지나간 일정은 삭제하지 않고 `text.tertiary`로 톤다운 + "완료" 캡션 — 하루의 진행률이 한눈에 보이는 타임라인 효과.

**세션별 우선 데이터**: 장전 = 개장 카운트다운 행이 시각 강조 / 장중 = 다음 이벤트가 자동으로 최상단 스크롤 위치(순서는 불변, 지난 항목만 톤다운) / 장후·야간 = 21:30 미국 지표·22:30 미국 개장 등 야간 이벤트 구간이 자연스럽게 "남은 일정"으로 부각.

## 5. 디자인 의도

1. **"결론 먼저, 숫자 다음, 맥락 그다음, 행동 마지막."** Hero(해석) → 지수(사실) → 뉴스(재료) → 일정(다음 행동)의 4단은 아침 의사결정의 인지 순서를 그대로 화면 위계로 옮긴 것이다. 사용자가 정보를 찾아다니는 게 아니라, 화면이 사고 순서대로 정보를 흘려보낸다.
2. **무스크롤 단일 화면은 기능이다.** 1080px 예산 역산(§1-1)으로 5개 카드를 스크롤 0에 전부 배치했다. "3분 파악"의 최대 적은 스크롤로 인한 재탐색 비용이며, 이를 없애기 위해 콘텐츠 내 h1 생략·카드 높이 고정 같은 트레이드오프를 의도적으로 감수했다.
3. **공간은 불변, 상태만 가변.** 장전/장중/장후에 카드를 재배열하지 않는다. 매일 같은 자리에서 같은 정보를 찾는 반복 사용자(하루 수 회 접속)의 공간 기억이 곧 속도이기 때문이다. 세션 변화는 라벨·라이브 점·톤다운이라는 "상태 문법"(§9-4)으로만 표현한다.
4. **신뢰는 타임스탬프에서 나온다.** 야간선물 오독 사고(P3·§9-4)의 교훈을 페이지 전면에 적용했다 — 모든 숫자 옆에 as-of, 세션 밖 라이브 연출 전면 금지, 등락률 기준가 병기. "이 숫자는 언제 것인가"에 답하지 못하는 픽셀은 이 페이지에 존재하지 않는다.
5. **Hero와 지수 카드의 역할 분리.** Hero의 숫자는 "리포트 생성 시점의 스냅샷"(문서), ②③의 숫자는 "지금 상태"(계기판)다. 같은 KOSPI가 두 번 등장하지만 캡션("07:12 기준" vs "● 장중")으로 역할이 갈리며, 이 중복은 낭비가 아니라 결론과 현황의 교차 검증 장치다.
6. **밀도는 그리드가 감당한다.** 한 화면에 지수 8종 + 뉴스 10건 + 일정 7건 + 리포트 요약이 담기지만, 4pt 스페이싱·tabular 숫자 우측 정렬·회색 아웃라인 배지 덕에 "빽빽하되 정돈된" Bloomberg적 밀도를 유지한다 — 복잡함의 해법은 색과 장식의 제거이지 정보의 제거가 아니다(§0-1).

## 6. 색상 및 여백 사용 이유

### 6-1. 색 — 의미 색 4종 예산의 실제 배분

이 페이지의 평상시 뷰포트에 존재하는 유채색 전수(R1 검증):

| 색 | 등장 위치 | 슬롯 |
|---|---|---|
| `market.up` #F87171 / `market.down` #60A5FA | 등락 텍스트·▲▼·스파크라인·갱신 플래시 12% | 등락 2색 |
| `brand.primary` #10B981 계열 | 사이드바 활성 인디케이터, Hero CTA 1개, `badge.live` 라이브 점 | CTA 1색 |
| `accent.violet` #A78BFA | Hero 한 줄 요약 좌측 바 3×20px **단 1곳** | 해석 레이어(면적 최소) |
| `accent.yellow` #FBBF24 | **평상시 부재.** 지연·스테일 배지 발생 시에만 등장 | 상태 1색(조건부) |

**페이지 규칙: 상태색(노랑)이 등장하는 프레임에서는 violet 바를 회색으로 강등한다** — "상태가 해석보다 우선"이며, 어떤 순간에도 동시 의미 색이 4종을 넘지 않는다. 나머지 구분(뉴스 카테고리, 일정 강조, 필터 탭)은 전부 회색 단계+보더+굵기로 처리(R2) — 그래서 화면에서 빨강/파랑이 보이면 무조건 등락이라는 단일 해석이 성립하고, 이것이 "많아 보이되 복잡하지 않음"의 실체다.

한국 모드 기본(상승 빨강/하락 파랑, §2-5)에서 브랜드 초록과 등락색이 완전 분리되므로, 라이브 점(초록)이 상승 신호로 오독될 여지도 구조적으로 없다. 등락 값은 항상 13px 이상 + 500 이상 + ▲▼ 병행(§11-1 대비 보완, R4).

### 6-2. 여백 — 24px 리듬과 밀도의 균형

- **거터 = 카드 간격 = 24px 단일 리듬**: 행·열 간격이 전부 24px이라 카드 5장이 하나의 판처럼 읽힌다. 섹션 간 32px을 쓰지 않은 이유는 이 페이지가 "여러 섹션"이 아니라 "하나의 브리핑 보드"이기 때문 — 간격 차등은 오히려 위계 없는 위계를 만든다.
- **카드 내부 패딩 24(Hero)/20(Standard)**: Hero만 4px 더 넉넉한 패딩으로 "가장 중요한 카드"임을 여백으로도 말한다. 크기(340px)·타이포(32px)·패딩(24px) 세 채널이 같은 방향을 가리키는 중복 위계.
- **라벨↔값 4px, 아이콘↔텍스트 8px, 표 셀 좌우 12px**: 4pt 스케일 최솟값들을 데이터 밀집 구역에 배정해, 큰 여백(24px)은 구조를, 작은 여백(4~12px)은 판독성을 담당하게 역할 분리.
- **깊이는 그림자 없이**: Level 1 평면 카드는 `surface.card`+1px 보더만(§4-3). 미니 타일도 별도 표면 없이 `border.subtle` 십자 디바이더만 사용 — 표면 단계를 아껴야 검색 패널·모달(Level 2·3)이 떴을 때의 대비가 산다.
- **숫자 정렬 여백**: 모든 수치 열은 tabular figures + 우측 정렬(§3-1)이라, 자릿수가 달라도 소수점·천 단위가 수직으로 맞는다. 이 정렬 자체가 여백 설계의 일부다 — 흔들리는 숫자 열은 어떤 여백으로도 구제되지 않는다.

## 7. 개선 포인트

1. **정적 환경의 신선도 판정 정밀화**: GitHub Pages 특성상 §9의 판정은 "빌드 as-of 타임스탬프 + 클라이언트 시계" 비교로 수행된다(§12). 데이터 파일에 값·생성 시각·유효 세션 메타를 항상 포함시키고, 소스별 기대 주기 T를 분리 운영한다 — 모닝리포트 T=24h(캘린더 기준), 뉴스 T=3h, 지수 스냅샷 T=배치 주기. "장중 라이브"는 실제로는 배치 갱신이므로, 배치 간격이 T를 넘기면 라이브 점 대신 DELAYED 배지로 정직하게 강등하는 것이 이 제품의 신뢰 전략이다.
2. **관심종목 Row 카드(선택 노출)**: 사용자가 종목을 등록하면 3행 아래 12col Standard 카드(행 52px, §8-4 스펙)로 추가. 기본은 미노출 — 무스크롤 원칙을 깨는 대신, 등록 시에만 "1스크롤 존"이 열리는 opt-in 구조.
3. **리포트 아카이브·전일 대비 diff**: Hero 헤더에서 날짜 네비게이션(← 07/09)으로 과거 리포트 열람, "어제 대비 달라진 문장"을 `accent.violet` 톤으로 하이라이트하는 해석 레이어 확장.
4. **일정 카드의 로컬 알림**: 브라우저 Notification API(로컬, 서버 불필요)로 ★핵심 이벤트 10분 전 알림. Settings에 기본 off 토글.
5. **글로벌 모드 QA 항목화**: 등락 색 전환(§2-5) 시 이 페이지의 스파크라인·갱신 플래시·▲▼ 병행 규칙이 일괄 전환되는지 상태 매트릭스(§3-7) 기준 회귀 체크리스트를 별도 문서화.
6. **1280px 미만 열람 모드**: 공식 지원 외 구간(§5-2)이지만, 모바일 출근길 수요를 고려해 Hero 한 줄 요약+지수 4타일만 남기는 "브리핑 축약 뷰"를 차기 버전에서 검토.
7. **빈 상태·에러의 실데이터 검증**: 리포트 생성 실패 시 Hero는 §9-3 원칙대로 전일 리포트를 STALE 스타일로 유지 + `status.danger` 배지(정보 제로 회피). 이 폴백이 실제 배치 실패 시나리오에서 작동하는지 빌드 파이프라인과 함께 점검한다.
