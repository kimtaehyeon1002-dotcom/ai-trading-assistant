# 04. Stock

**AI Trading Assistant — Stock 페이지 설계 문서 v1.0**
기준: 글로벌 디자인 시스템(00) v1.0의 토큰·수치를 변경 없이 인용. Desktop First 1920×1080, 다크 테마 단일.
페이지 정의: **테마 스크리너 + 거래대금 랭킹 허브** — "오늘 시장의 돈이 어디로 흐르는가"를 3초 안에 답하고, 종목 단위로 파고들 수 있는(Drill-down) 진입 관문.

## 1. 전체 레이아웃 설명

### 1-1. 페이지 골격

- 사이드바(240px, 좌측 고정): **Stock 메뉴 활성** — 좌측 인디케이터 바 3×20px `brand.primary`, 아이콘 `brand.primary`, 라벨 `text.primary` 600 (§6-2 그대로).
- 헤더(64px): 페이지 컨텍스트는 `Stock / 테마 스크리너`(`type.h3` + 브레드크럼 12px `text.tertiary`). Stock Hub 패널은 페이지 이동이 아닌 오버레이 컴포넌트이므로, 패널이 열리고 닫히는 동안 이 브레드크럼은 **변경되지 않는다**(05 「Stock Hub」§1-6이 정의하는 전역 계약 — 단일 진실 소스). 브레드크럼이 실제로 갱신되는 시점은 Financial Statements 등으로 **실제 페이지 이동**이 발생할 때뿐이다.
- 콘텐츠 영역: 최대폭 1600px, 좌우 마진 32px, 중앙 정렬, 12컬럼 그리드 / 거터 24px. 헤더 아래 상단 여백 24px. 이하 본 문서의 카드 폭 환산: **12col ≈ 1600px, 6col ≈ 788px, 1col ≈ 112px**.

### 1-2. 콘텐츠 수직 구성 (위 → 아래)

| 순서 | 영역 | 스팬 × 높이 | 역할 |
|---|---|---|---|
| ① | 컨트롤 바 (sticky) | 12col × 56px | 종목 필터 검색 · 테마 선택 칩 · 국가 세그먼트 · as-of 시각 |
| ② | 테마 자금 흐름 요약 스트립 | 12col × 72px | 페이지의 "결론 요약" — 오늘 거래대금 상위 테마 5개 (P5 충족) |
| ③ | 테마별 스크리너 카드 그리드 | 6col × 360px, 2열 × 3행 | 테마 6종(반도체·AI·방산·원전·2차전지·로봇) 각 한국 TOP5 + 미국 TOP5 |
| ④ | 거래대금 TOP30 테이블 ×2 | 각 6col × 768px 병렬 | 한국 TOP30(좌) / 미국 TOP30(우), 내부 스크롤 + 고정 헤더 |
| — | Stock Hub (오버레이) | 우측 480px 슬라이드 패널 | 종목명 클릭 시 열리는 전역 컴포넌트 (§3-5에서 이 페이지에서의 호출 계약만 정의) |

섹션 간 수직 간격 32px, 카드 간 간격 24px (§4-1).

### 1-3. 2단 구조 (3초 파악 → 파고들기)

- **3초 파악**: 스크롤 0 상태에서 요약 스트립("오늘 자금 상위 테마: 반도체 3.2조…") + 첫 테마 카드 행이 보인다. 시선은 좌상(요약) → 우하(카드)의 Z-패턴.
- **파고들기**: 테마 칩 필터 → 카드 확장(TOP10) → 종목 클릭 → Stock Hub → TradingView 차트(외부 새 탭) 또는 Financial Statements(내부 이동)로 분기. 어떤 경로든 3클릭 이내 (§4에서 검증). Stock Hub의 액션 버튼 구성은 05(Stock Hub) §3-4 마스터 스펙에 정확히 2개로 고정되어 있으며, 이 페이지에서 임의로 추가·변경하지 않는다(05 §5).

### 1-4. 컨트롤 바 동작 정의 (상단 요구사항: 검색창·테마 선택·국가 선택)

- **종목 필터 검색** (320×36px, 배경 `surface.sunken`, 보더 `border.default`, radius 8px): 헤더의 글로벌 검색(360px, Ctrl K)과 역할 분리 — 이 필드는 **현재 페이지 내 필터**다. 플레이스홀더 "이 페이지에서 종목 필터 (이름·티커)", `text.tertiary`. 입력 즉시 TOP30 테이블은 일치 행만 남기고, 테마 카드의 불일치 종목 행은 `text.disabled` 톤으로 딤 처리(레이아웃 점프 금지). Enter 시 첫 일치 종목의 Stock Hub 오픈. 일치 0건이면 테이블 본문에 빈 상태(§9-3): "일치하는 종목이 없습니다 — 전역 검색으로 찾기 (Ctrl K)".
- **테마 선택 칩** (높이 32px, 좌우 패딩 12px, `radius.badge` 999px, 칩 간격 8px): `전체 · AI · 반도체 · 방산 · 원전 · 2차전지 · 로봇 · 조선` 단일 선택. 기본 상태 `border.default` 아웃라인 + 라벨 13px/500 `text.secondary`, 호버 `surface.card-hover`, 활성 `surface.selected`(#10B981 8% 오버레이) + 라벨 `text.primary` 600 + 보더 `brand.primary` 1px. 칩이 넘치면 가로 스크롤 + 우측 24px 페이드 + `›` 힌트.
- **테마 칩 선택 시 동작**: ① 테마 카드 그리드가 해당 테마 카드 1장만 **12col 확장 모드**(한국 TOP10 + 미국 TOP10)로 전환 ② TOP30 테이블은 해당 테마 종목 행만 유지(숨겨진 행은 접힘, **순위 숫자는 전체 기준 유지** — 필터 중임이 순위의 불연속으로 자연히 드러남). "전체" 칩으로 복귀.
- **국가 세그먼트** (240×36px, 3분할 각 80px, radius 8px, 배경 `surface.sunken`): `전체(기본) / 한국 / 미국`. 활성 세그먼트는 `surface.card` 배경 + `text.primary` + `border.default`, 비활성은 `text.secondary`. `한국` 선택 시: 테마 카드는 한국 TOP5가 카드 전폭 사용(거래대금·스파크라인 컬럼 추가), 미국 절반 숨김; 한국 TOP30이 12col로 확장(§3-4), 미국 테이블 숨김. `미국`은 대칭.
- **요약 스트립은 국가 필터 범위 밖**: ② 테마 자금 흐름 요약 스트립(§3-2)은 국가 세그먼트 선택과 **무관하게** 항상 한·미 합산(전체 시장) 기준을 유지한다 — `한국`/`미국`으로 좁혀도 스트립 내용은 필터링되지 않는다. 스트립 우측에 "전체 시장 기준" 캡션을 상시 병기해, 하단 카드·테이블(필터된 범위)과 상단 결론(항상 전체 범위)이 다른 모집단을 가리킬 수 있음을 명확히 한다.
- **컨트롤 바 sticky**: top 64px(헤더 바로 아래) 고정, 배경 `surface.base` 불투명, 하단 `border.subtle` 1px. 스크롤 발생 시 하단 보더가 `border.default`로 강화되어 "떠 있음"을 표현(그림자 금지 — 같은 평면, §4-3).
- 우측 끝: as-of 캡션 `type.caption` `text.tertiary` — "기준 07/10 09:40 KST · 20분 주기 갱신". 클릭 시 데이터 신선도 패널(§9) 열림.

### 1-5. 세션 인식 렌더링 적용 (§9-4)

- 한국 종목: KST 09:00–15:30 장중이면 FRESH 표기 + 라이브 점(`badge.live`, 2s 펄스) 허용. 그 외 시간은 CLOSED-SNAPSHOT — "07/10 15:30 마감 기준" 라벨, 라이브 점·펄스 전면 금지.
- 미국 종목: KST 주간에는 항상 전일 마감 스냅샷 — 모든 미국 카드·테이블 헤더에 "**07/09 마감 기준(ET)**" 라벨 필수(생략 불가). 값은 `text.primary` 유지(마감 스냅샷은 STALE이 아님).
- 등락률 기준가 병기(S3): 한국 = "전일 종가 대비", 미국 = "전일 정규장 종가 대비". Stock Hub 내에 명시.

## 5. 디자인 의도

1. **"돈의 흐름"을 페이지의 결론으로**: 이 페이지의 결론은 개별 종목이 아니라 **자금이 몰린 테마의 순위**다. 그래서 P5에 따라 스크롤 0 최상단에 요약 스트립을 놓고, 개별 종목(테마 카드)과 전수 랭킹(TOP30)을 그 아래 위계로 배치했다. "요약 → 테마 → 종목"의 3층 깔때기 구조.
2. **랭킹의 근거를 항상 보이게**: 모든 리스트가 거래대금 기준이므로, 거래대금을 숨기지 않고 테마 카드 행(2줄째)·TOP30(데이터 바)에 상시 노출했다. 정렬 기준이 보이지 않는 랭킹은 신뢰를 잃는다. TOP30의 회색 데이터 바는 Bloomberg식 밀도를 색 추가 없이 얻는 장치다.
3. **순위 고정 원칙**: 정렬 토글로 순서를 바꿔도 순위 숫자는 거래대금 기준으로 고정하고 dim 처리했다. "순위"가 정렬에 따라 재계산되면 순위라는 단어의 의미가 매번 바뀌어 스크린샷·기억 기반 대화("오늘 3위였던 한화에어로")가 불가능해진다.
4. **한·미 병렬 대칭**: 한국(좌)·미국(우)을 카드 내부와 TOP30 모두에서 동일한 좌우 배치로 통일 — 한 번 학습한 공간 문법이 페이지 전체에서 재사용된다. 단위 표기는 시장별 규칙(₩/조·억 vs $/B)을 컴포넌트 단위로 분리해 혼합 금지(§3-3).
5. **세션 진실성(P3·§9-4)**: 미국 데이터는 한국 아침에 항상 "전일 마감 기준"이라는 사실을 디자인이 먼저 말한다. 미국 열·테이블에 마감 기준 라벨을 구조적으로 강제해, 야간선물 사고와 같은 "마감값의 라이브 오독"을 이 페이지에서 원천 차단했다.
6. **테이블이 곧 제품**: 이 페이지 체류 시간의 대부분은 TOP30 스캔이다. 40px 밀집 행 × 16행 노출(한 화면에 정보 32종목분), tabular 우측 정렬, sticky 헤더, 3단 정렬 순환, 행 전체 클릭 — 트레이딩 테이블의 기본기를 px 단위로 못 박아 어떤 구현에서도 흔들리지 않게 했다.
7. **Stock Hub는 비모달**: 종목 확인은 "리스트 탐색의 중단"이 아니라 "탐색 중의 곁눈질"이다. 딤 없는 사이드 패널로 리스트 ↔ 종목 사이를 클릭 1회로 오가게 했고, 트리거 행의 `surface.selected` 앵커가 맥락 상실을 방지한다.

## 6. 색상 및 여백 사용 이유

### 색상

- **의미 색 4종 상한(R1) 검증**: 이 페이지 뷰포트의 유채색은 ① `market.up` #F87171 ② `market.down` #60A5FA ③ `accent.yellow`(마감 기준/지연 배지) ④ `brand.primary`(사이드바 활성·칩 활성·Stock Hub CTA) — 정확히 4종으로 상한에 걸쳐 있다. 따라서 이 페이지에서는 `status.info` 배너, `accent.violet` 인사이트 등 5번째 색을 추가하지 않는다.
- **테마 배지 = 회색 아웃라인**: 테마 6종을 색으로 구분하면 6색이 등락 2색과 뒤섞여 "빨간 배지 = 상승?"이라는 오독이 생긴다(P2 실제 작동 예 그대로). 구분은 `type.overline` 타이포 + 아웃라인 형태로 충분하다.
- **거래대금 데이터 바 = `skeleton.base` 단일 회색**: 크기는 의미이지만 방향이 아니므로 유채색이 필요 없다. 회색 바는 R2(회색 단계 우선)의 정석 적용이며, 등락 색의 시각적 발언권을 침해하지 않는다.
- **절대값 흰색 / 변화값만 등락색**: 현재가·거래대금은 `text.primary`, 등락률만 `market.*` — 색이 붙은 숫자는 "방향 정보"라는 단일 규칙이 표 전체에서 유지된다. ▲▼ 기호 병행(R4)으로 색맹 사용자에게도 방향이 1차 채널로 전달된다.
- **`surface.selected`의 단일 용도**: Stock Hub 트리거 행에만 사용하고 호버(`surface.card-hover`)와 명확히 분리 — "지금 열려 있는 것"과 "지금 스치는 것"의 혼동 금지(§2-1 금지 사례 준수).

### 여백

- **행 높이의 2단계 운용**: 테마 카드 52px(2줄, 탐색·발견용 — 여유가 가독을 만든다) vs TOP30 40px(1줄, 전수 스캔용 — 밀도가 효율을 만든다). 같은 페이지에서 용도별로 Row 등급(§8-2)을 갈라 쓴 것.
- **4pt 스케일 준수**: 카드 패딩 20px(Standard)/16px(Compact 스트립), 카드·거터 24px, 섹션 간 32px, 라벨–값 4px, 셀 좌우 12px — 임의 값 없음. 균일한 리듬이 "정보는 많지만 복잡하지 않다"는 인상의 실체다.
- **여백이 곧 구분선**: 테마 카드 내부 종목 행은 구분선 없이 52px 행높이의 여백만으로 분리했다. 반대로 40px 밀집 테이블은 `border.subtle` 1px 구분선이 필요하다 — 밀도가 높아질수록 보더가 여백을 대신한다는 원칙(P4)의 페이지 내 적용.
- **16행 노출 × 내부 스크롤**: TOP30을 전체 렌더(1,288px)하면 페이지 스크롤이 테이블에 잠식되고 미국 테이블과의 병렬 비교가 깨진다. 768px 고정 + sticky 헤더로 두 테이블의 세로 정렬을 유지, 좌우 눈 이동만으로 한·미 비교가 가능하다.

## 7. 개선 포인트

1. **정적 빌드 갱신 주기와 신선도 판정**: 서버 없는 GitHub Pages 특성상 "장중 20분 주기"는 배치(Actions) 주기에 의존한다. 데이터 파일에 as-of·유효 세션 메타를 포함시키고(§12), 클라이언트 시계 비교로 FRESH/DELAYED/STALE을 판정한다. 정렬·필터·검색은 전량 클라이언트 사이드로 처리 가능(30행 × 2 규모라 가상 스크롤 불필요).
2. **테마 분류의 투명성**: 한 종목이 복수 테마(예: 팔란티어 = AI·방산)에 걸칠 때 대표 테마 1개만 배지로 노출하고, Stock Hub에 전체 테마를 표기하는 규칙이 필요하다. 향후 테마-종목 매핑 편집 UI(Settings 하위)를 검토.
3. **테마 히트맵 뷰**: 요약 스트립을 확장한 트리맵(테마 × 거래대금 면적 × 등락 색) 토글 뷰. 단, 등락 2색 면적이 커지면 R1 색 예산과 충돌하므로 명도 단계 기반 설계 선행 필요.
4. **관심종목 연동**: TOP30 행 좌측에 즐겨찾기 별(`accent.yellow`, 히트 32px) 추가 → Dashboard 관심종목 카드와 동기화. 색 예산상 accent.yellow의 3번째 용도가 되므로 배지와의 시각 구분(채움 vs 아웃라인) 규칙 필요.
5. **랭킹 변화 표시**: 전일 대비 순위 등락(↑3, ↓2, NEW)을 순위 컬럼 옆 11px `text.tertiary`로 병기 — "오늘 새로 돈이 들어온 종목" 발견 가치가 크다. 배치 데이터에 전일 순위 스냅샷 포함이 전제.
6. **Stock Hub 딥링크**: 패널 상태를 URL에 반영해 새로고침·공유 시 동일 종목 패널 복원. 브레드크럼·브라우저 뒤로가기와의 일관 동작 설계 필요.
7. **1280px 접힘 대응 상세화**: 사이드바 자동 접힘 시 테마 카드 6col → 12col 순차 스택, TOP30 두 테이블도 세로 스택(한국 우선). 이때 한·미 병렬 비교가 깨지므로 국가 세그먼트의 기본값을 "한국"으로 바꾸는 축소 전용 규칙을 검토(§5-2 스팬 축소 우선순위 준수).
8. **거래대금 외 랭킹 축**: 거래량 급증률, 외국인 순매수 등 랭킹 기준 전환 드롭다운. 단, "TOP30 = 거래대금"이라는 페이지 정체성이 흐려지지 않도록 별도 탭이 아닌 카드 헤더 [⋯] 메뉴 수납을 우선 검토.
