# Obsidian Vault — 장기기억 저장소

이 폴더를 Obsidian에서 vault로 연다. git(obsidian-git 플러그인)으로 동기화하며,
GitHub Actions 빌드가 이 폴더를 읽고(워치리스트) 쓴다(저널·메모리).

## 쓰기 주체 규칙 (git 충돌 원천 차단)

| 폴더 | 쓰는 주체 | 내용 |
|---|---|---|
| `00_Watchlist/` | **사용자만** | 관심종목 (1노트 = 1종목, frontmatter가 데이터) |
| `10_Journal/` | **봇만** | 모닝리포트·뉴스·매매일지 요약 (빌드가 자동 생성) |
| `20_Memory/` | 봇 생성 + 사용자 가필 | 종목·테마별 누적 노트 (백링크 허브) |
| `90_Templates/` | 사용자 | Obsidian 템플릿 |

봇은 `20_Memory/`에서 **없는 파일만 스텁 생성**하고 기존 파일은 절대 수정하지
않는다 → 사용자 가필이 보호된다.

## 프라이버시 규칙 (v2 리디자인과의 호환 계약)

- **자산·현금흐름·목표금액은 이 vault에 두지 않는다.** v2 설계(design/22-architecture.md §7)가
  자산 파이프라인을 "4계좌 API 전량 자동수집 → 암호화(`assets.enc.json`) 발행"으로 확정했고,
  절대금액·역산 가능 값의 평문 커밋을 금지한다. vault는 리포에 평문으로 커밋되므로
  이 데이터의 자리가 아니다.
- 워치리스트(종목명·티커·목표가·메모)는 투자 의향 메모 수준으로, v2 유니버스 구성
  (설계 22 §9-1: TOP30×2 ∪ 테마 종목 ∪ watchlist)의 입력이 된다. 리포가 공개라면
  워치리스트도 공개됨을 인지하고 기록할 것.

## 워치리스트 노트 형식

frontmatter 키는 대시보드/유니버스가 소비하는 계약이다 (`90_Templates/워치리스트.md` 참조):

```yaml
---
종목명: 삼성전자
티커: "005930"
시장: KRX
목표가: 90000        # 선택
메모: 반도체 업황 회복  # 선택
---
```

본문은 자유 메모 영역 — 파싱하지 않는다. `[[20_Memory/stocks/삼성전자]]`처럼
메모리 노트로 링크를 걸면 백링크로 기억이 연결된다.
