## 1. 전체 레이아웃 설명

### 1-1. 페이지의 역할과 사용 맥락

Macroeconomics는 "오늘 시장의 날씨"를 확인하는 페이지다. 주 사용 시점은 세 가지로 명확하다.

1. **출근 전 장전 브리핑(07:30~08:50, 체류 1~3분)** — 밤사이 미국시장·야간선물·환율·원자재 확인. 전체 트래픽의 과반.
2. **장중 급변 시 원인 탐색(09:00~15:30, 체류 30초~2분)** — 변동성·환율·금리 그룹으로 직행.
3. **저녁 복기 + 미국 지표 발표 확인(18:00~23:00, 체류 2~5분)** — 야간선물 세션 시작, 경제지표 발표치 대조.

세 맥락 모두 "특정 그룹으로 즉시 점프"가 핵심 행동이므로, 이 페이지의 구조적 해법은 **① 최상단 Hero 요약(P5) + ② Sticky 앵커 내비게이션 + ③ 균일한 그룹 섹션 반복 패턴**이다.

- 콘텐츠 최대폭 1600px, 좌우 마진 32px, 12컬럼 그리드 / 거터 24px(00 §5-1 그대로).
- 섹션 간 수직 간격 32px(단, 한국시장 섹션은 랜드마크 처리로 상하 40px), 섹션 타이틀(`type.h2` 20px/600)과 첫 카드 사이 16px.
- 각 섹션 타이틀 우측에는 **그룹 공통 as-of 캡션**(`type.caption` 12px, `text.tertiary`)을 상시 표기한다. 예: "07/10 05:00 마감 기준".
- 페이지 총 높이 약 3,900px — 스크롤 4~5 뷰포트 분량. 그래서 앵커 바가 필수 장치다.

### 1-3. 앵커 내비게이션 바 (이 페이지의 핵심 신규 컴포넌트)

| 속성 | 값 |
|---|---|
| 위치 | `position: sticky`, 헤더 아래(top 64px)에 고정. 배경 `surface.base` |
| 높이 | 48px, 스크롤 진입 후 하단에 `border.subtle` 1px 표시(정지 상태에서는 보더 없음) |
| 칩(앵커 항목) | 높이 32px, 좌우 패딩 12px, `radius.control` 8px, 라벨 13px/500 `text.secondary`, 칩 간 간격 8px |
| 호버 | 배경 `surface.card-hover`, 라벨 `text.primary`, `motion.fast` 120ms |
| 활성(스크롤 스파이) | 배경 `brand.primary` 10% 오버레이, 라벨 `text.primary` 600 — 사이드바 활성 메뉴(00 §6-2)와 동일 문법 |
| 항목 수 | 9개: 미국시장 · 한국시장 · 환율 · 채권·금리 · 원자재 · Bitcoin · 변동성 · 경제지표 · 경제일정 |
| 우측 끝 | "↑ 맨 위로" 고스트 버튼(28×28px, 히트 영역 32px) — 1뷰포트 이상 스크롤 시에만 표시 |
| 동작 | 클릭 시 해당 섹션으로 스크롤(오프셋 128px = 헤더 64 + 앵커 바 48 + 여백 16, `motion.base` 200ms). 스크롤 스파이는 뷰포트 상단 128px 라인 통과 기준으로 활성 갱신 |
| 키보드 | Tab으로 바 진입 → `←→`로 칩 이동 → Enter 점프. `focus.ring` 2px #34D399 적용 |
| 배지 | "경제일정" 칩에는 오늘 이벤트가 있으면 우상단 6px `accent.yellow` 도트. "한국시장" 칩에는 **정규장(09:00–15:30) 또는 야간선물 세션(18:00–05:00) 중 하나라도 활성 상태**이면 6px `badge.live` 도트(2s 펄스)를 표시 — 두 세션 모두 비활성인 구간(05:00~09:00, 15:30~18:00)에만 도트를 끈다 |

9개 칩 총폭 약 900px로 1600px 콘텐츠 폭 내 한 줄 수용. 색이 아니라 밝기·굵기로 활성을 표현하므로 R1(의미 색 4종 제한)을 침범하지 않는다. **[리뷰 반영]** 한국시장 칩의 라이브 조건은 원래 야간선물 세션에만 걸려 있었으나, 정작 국내 지수가 가장 활발히 움직이는 정규장 시간대에는 앵커 바가 침묵하는 모순이 있었다 — "지금 어디에 움직임이 있는지" 알려준다는 §5 ⑥의 설계 의도를 실제로 만족하려면 하루 중 가장 중요한 구간을 빠뜨려서는 안 되므로 조건을 확장했다. `badge.live`는 헤더 듀얼 시계(00 §7-3)가 이미 정규장·야간선물 두 세션 모두에 쓰는 동일 토큰이라 새 색을 추가하지 않는다.

### 1-4. 사이드바·헤더 특이사항 (00에서 정의된 부분은 생략)

- 사이드바: **Macroeconomics 항목 활성** — 좌측 3×20px `brand.primary` 인디케이터 바, 아이콘 `brand.primary`, 라벨 `text.primary` 600 (00 §6-2 그대로).
- 헤더 좌측 페이지 컨텍스트: "Macroeconomics" 단독 표기(`type.h3`), 브레드크럼 없음(최상위 페이지).
- 헤더 듀얼 시계(00 §7-3)는 이 페이지에서 특히 중요 — 시계 클릭 팝오버의 "야간선물 세션 상태"가 본문 한국시장 그룹의 세션 상태, 그리고 앵커 바 "한국시장" 칩의 라이브 도트(§1-3)와 항상 동일한 판정 로직을 공유한다(단일 세션 판정기, §9-4 S1). 정규장(09:00–15:30)·야간선물(18:00–05:00) 중 하나라도 활성이면 헤더 시계·앵커 칩·본문 카드 세 곳이 동시에 라이브 상태로 전환된다 — 어느 한 곳만 갱신되고 나머지가 뒤처지는 것을 막기 위해 반드시 같은 판정 함수를 참조한다.

### 3-2. 카드별 내부 구성과 정보 우선순위

**① 매크로 한 줄 요약 Hero (8col, 패딩 24px)**
- 1순위: 요약 문장 — `type.h2` 20px/600 `text.primary`, 2줄 이내. 문장 앞에 `text.tertiary` 톤의 "◆" 마커 1개만 두어 문장의 시작점을 표시한다. **[리뷰 반영]** 이전 버전은 좌측에 `accent.violet` 3×40px 인디케이터 바를 두어 "해석 레이어"임을 색으로 계약했으나, 동일한 "좌측 3px violet 바 + 회색 텍스트" 조합이 Dashboard·News·Financial Statements·Asset 등 5개 이상 페이지에서 반복된다는 지적에 따라 이 페이지에서는 제거했다. `accent.violet`의 "정식" 좌측 바 형태는 제품에서 가장 응축된 판단이 필요한 **Dashboard·Financial Statements 두 곳에만** 남기고(00 §2-3-1), Macro를 포함한 나머지 페이지는 기본값인 무채색 처리로 되돌린다는 것이 00 문서에 정식 등재된 규칙이다. 이 Hero의 특별함은 이제 색이 아니라 **위치(페이지 최상단)·타이포 크기(h2 20px, 이 페이지 본문 텍스트 중 가장 큼)·여백(패딩 24px, 카드 중 최대)** 세 가지 구조적 신호만으로 전달한다(P4 위계는 밝기로).
- 2순위: 핵심 지표 스트립 6종 — 값 `type.num-md` 16px `text.primary`, 등락 `type.num-sm` 13px `market.*`, 열 간격 40px, tabular 우측 정렬. 각 지표 클릭 시 해당 그룹 섹션으로 앵커 점프.

## 5. 디자인 의도

**① "그룹 반복 패턴"으로 학습 비용을 0에 수렴시킨다.** 9개 그룹 모두 [h2 타이틀 + as-of 캡션 → 카드 행]의 동일 골격을 반복한다. 사용자는 첫 그룹에서 읽는 법을 익히면 나머지 8개를 같은 눈으로 읽는다. Bloomberg의 정보 밀도를 유지하면서 복잡해 보이지 않게 하는 방법은 화려한 구획이 아니라 **지루할 만큼 균일한 리듬**이다. 단, 이 균일함이 스크롤 탐색의 리듬 감각까지 없애서는 안 된다 — 그 균형점은 ②에서 다룬다.

**② 균일한 리듬 속에 두 개의 정지점을 심는다 — 랜드마크 설계(리뷰 반영).** ①의 '지루할 만큼 균일한 리듬'은 학습 비용을 낮추지만, 그 대가로 Hero를 지나는 순간부터 스크롤로 훑어보는 사용자에게 "무엇이 더 중요한지" 알려주는 시각적 랜드마크가 사라진다는 지적이 있었다. 앵커 바는 '점프'해서 찾는 사용자에게는 해법이지만, 그냥 스크롤을 내리며 훑는 사용자에게는 아무 도움이 안 된다. 9개 그룹 전부를 차등화하면 ①의 장점이 사라지므로, 사용 빈도가 가장 높은 딱 두 곳만 예외로 둔다. ⓐ **한국시장**(§3-2) — 국내 사용자가 가장 자주 참조하는 섹션이자 야간선물 사고 재발 방지 장치(④)가 있는 곳이라 스크롤 초반(9개 섹션 중 2번째)에 시선을 붙잡아야 한다. 섹션 전체를 `border.default` 1px + `radius.card` 12px 아웃라인 패널로 감싸 나머지 7개 섹션과 구분하되, 배경은 채우지 않아 카드의 `surface.card`와 겹치지 않게 한다. ⓑ **경제지표**(9번째 섹션) — 이미 12col 풀폭 테이블이라는 형태 자체가 나머지 카드형 섹션과 다르므로 별도 처리 없이 그 이질성을 랜드마크로 승격한다. 두 지점은 각각 스크롤 초반부와 후반부에 위치해 3,900px 여정에 리듬을 다시 환기시키는 두 번의 '정지 신호'를 준다. 카드 등급(Compact/Standard) 자체를 키우지 않고 섹션 레벨의 보더·여백 차이로만 해결했다는 점에서 R2(장식적 유채색 금지)와 00의 카드 등급 규약을 함께 지킨다.

**③ Hero의 주인공을 숫자가 아닌 '문장'으로 둔 유일한 예외 — 단, 색이 아니라 구조로 증명한다.** P1(Numbers First)은 페이지 전체에 관철되지만, Hero만은 해석 문장이 1순위다. 이 페이지의 숫자는 아래 27개 카드에 이미 충분하고, 사용자가 아침에 원하는 것은 "그래서 오늘 어떻다는 건가"라는 결론이기 때문이다. 사실(숫자)과 해석(문장)의 경계는 `accent.violet` 같은 색이 아니라 **위치·타이포 크기·여백**으로 긋는다(§3-2 ①). 좌측 violet 바를 쓰지 않는 이유는 Dashboard·News·Financial Statements·Asset 등 최소 5개 페이지에서 동일한 "좌측 3px violet 바 + 회색 텍스트" 조합이 반복되어 "시스템이 짚어준 특별한 판단"이라는 신호력이 옅어졌다는 지적 때문이다(00 §2-3-1). violet 해석 레이어의 "정식" 형태는 제품에서 가장 응축된 판단이 필요한 Dashboard·Financial Statements 두 곳에만 남기고, Macro를 포함한 나머지 페이지는 기본값인 무채색 처리로 되돌려 색의 신호력을 지킨다. 그 결과 이 페이지가 실제로 쓰는 의미 색은 `market.up`/`down`·`accent.yellow`·`brand.primary` 3종뿐이다(§6) — R1의 4색 상한에 여유가 남는 것은 절제 부족이 아니라, 27개 카드가 몰린 페이지에서 5번째 색을 켜는 순간 신호 대 잡음비가 무너진다는 판단의 결과다.

**④ 야간선물 사고의 재발 방지를 '스타일'이 아니라 '카드 해부학'으로 해결한다.** 마감 라벨을 작게 붙이는 소극적 처방이 아니라, 세션형 Compact라는 별도 등급(+36px)을 만들어 기준가 병기 행과 세션 카운트다운 행을 구조적으로 상주시켰다. "다음 세션 18:00 개장 (10시간 20분 후)"라는 문장은 지금 보는 값이 과거임을 미래형 문장으로 한 번 더 증언한다 — 라벨은 안 읽어도 카운트다운은 읽힌다.

**⑤ 경제지표는 '표'가 정답이다.** 발표치/예상치/이전치의 3값 비교는 카드 분산 시 비교 자체가 불가능해진다. 12col 풀폭 테이블 + tabular 숫자 우측 정렬로 세로 스캔(한 지표의 3값)과 가로 스캔(여러 지표의 서프라이즈)을 동시에 지원한다. 유일한 흰색을 발표치 열에만 부여한 것은 "실제 세계가 내놓은 숫자"와 "사람들의 추측(예상치)"의 인식적 지위 차이를 밝기로 번역한 것이다.

**⑥ 앵커 바는 내비게이션이면서 상태 표시기다 — 그래서 '침묵'이 있으면 안 된다(리뷰 반영).** 한국시장 칩의 라이브 도트는 애초 야간선물 세션(18:00–05:00)에만 켜지도록 설계됐으나, 이는 국내 지수가 실제로 가장 활발히 움직이는 정규장(09:00–15:30)을 빠뜨리는 결과였다 — 하루 중 가장 중요한 시간대에 정작 내비게이션이 조용해지는 모순이었다(§4-3에서 실제 시나리오로 검증). 조건을 "정규장 또는 야간선물, 둘 중 하나라도 활성"으로 넓혀 이 공백을 없앴다(§1-3). 경제일정 칩의 이벤트 도트와 함께, 이제 앵커 바는 스크롤하지 않아도 "지금 어디에 움직임이 있는지"를 하루 종일 빠짐없이 알려주는 목차 겸 계기판이다.

**⑦ 3초 → 파고들기의 2단 구조.** 첫 화면 = Hero 결론 + 미국시장 + 오늘 일정(3초 파악). 파고들기 = 앵커 점프(1클릭) → 카드 클릭 히스토리 팝오버(2클릭). 어떤 매크로 정보도 페이지 진입 후 2클릭, 앱 전체 기준 3클릭 이내다(P5).

## 6. 색상 및 여백 사용 이유

**색 예산(R1) 배분 — 한 뷰포트 의미 색 4종 상한의 실제 운용:**

| 색 | 이 페이지의 용도 | 배제한 대안 |
|---|---|---|
| `market.up` #F87171 / `market.down` #60A5FA | 등락 텍스트·▲▼·스파크 틴트에만 | 카드 배경 등락색 칠하기 — 27개 카드가 있는 페이지에서 배경 등락색은 화면을 신호등으로 만든다 |
| `accent.yellow` #FBBF24 | STALE·지연 배지, 오늘/임박 일정 도트 | 일정 중요도 표시 — 중요도는 도트 개수(회색)로 대체. 노랑은 "시간 관련 주의" 단일 의미로 고정 |
| `accent.violet` #A78BFA | **이 페이지에서는 사용하지 않음(0곳, 리뷰 반영)** — Hero 요약 문장도 색이 아닌 타이포·여백·위치로 우선순위를 표현(§3-2 ①) | 좌측 바 + 회색 텍스트의 "해석 레이어" 박스 — Dashboard·Financial Statements 두 곳에만 남기기로 한 결정(00 §2-3-1)에 따라 이 페이지는 후보에서 제외한다. 색 예산이 남았다는 이유만으로 재도입하지 않는다 |
| `brand.primary` #10B981 | 사이드바 활성, 앵커 활성 오버레이 | 이 페이지에는 Primary 채움 CTA가 0개다(R3의 "최대 1개"보다 더 절제) — 열람 전용 페이지에 CTA는 존재 이유가 없다 |

이 페이지가 실제로 켜는 의미 색은 `market.up`/`down`·`accent.yellow`·`brand.primary` 3종뿐이다(`badge.live`는 `brand.primary` 계열 하이라이트를 재사용하므로 별도 색으로 세지 않는다). R1(4종 상한)에 1종 여유가 남지만, 이는 절제 부족이 아니라 "27개 카드가 몰린 페이지에서 5번째 의미 색을 켜는 순간 신호 대 잡음비가 무너진다"는 판단의 결과다 — 여유분은 §7의 향후 확장(상관관계 매트릭스 등)에서도 기본적으로 켜지 않는다.

**경제지표 서프라이즈를 무채색으로 결정한 이유(핵심 판단):** CPI 예상 상회는 시장에 악재, 고용 예상 상회는 호재가 될 수 있다 — 서프라이즈의 "방향"과 시장 반응의 "방향"이 지표마다 뒤집힌다. 여기에 `market.*` 색을 쓰면 "빨강 = 상승 = 좋음?"의 삼중 오독이 발생한다. 따라서 서프라이즈는 회색 아웃라인 배지 + 부호 + 텍스트(상회/하회/부합)로만 전달하고, 색은 시장 방향이 확정된 곳(등락)에만 남긴다. P2(Color = Meaning)의 가장 엄격한 적용 사례다.

**STALE의 밝기 강등이 배지보다 먼저다:** 오래된 값의 1차 처리는 노란 배지가 아니라 **값 자체를 `text.secondary`로 떨어뜨리는 것**이다(00 §9-2). 배지는 안 볼 수 있어도 "흐려진 큰 숫자"는 전주의적(pre-attentive)으로 감지된다. 야간선물 사고의 본질이 "흰색 큰 숫자의 권위"였으므로, 해법도 그 권위의 박탈이어야 한다.

**여백 운용:** 섹션 간 32px > 카드 간 24px > 카드 내 요소 간 4~16px의 3단 위계로, 보더나 배경색 없이 간격만으로 "그룹 > 카드 > 요소"의 소속을 전달한다. Compact 내부 패딩 16px·Standard 20px·Hero 24px의 차등은 카드 등급을 여백으로도 식별하게 한다. 표 행 52px(터치·마우스 겸용 여유)과 일정 행 40px(밀집)의 구분은 "비교하며 읽는 표"와 "훑는 리스트"의 읽기 속도 차이를 반영한 것이다. 스파크라인을 96×28px로 통일한 것은 카드 간 진폭 비교 오독(축 스케일이 다른데 모양만 비교하는 실수)을 줄이기 위해 크기라도 균일하게 고정하는 조치다 — 스파크라인에는 축·수치를 붙이지 않으므로 "모양 힌트" 이상의 지위를 주지 않는다. **[리뷰 반영]** 한국시장 섹션만 상하 여백을 32px→40px로 확장한 것도 같은 위계 논리의 연장이다 — 랜드마크 섹션이라는 정보를 색이 아니라 여백의 '양'으로 한 번 더 강조한다(§5 ②).

## 7. 개선 포인트

1. **스파크라인 기간 토글** — 현재 고정(지수 1일/환율 30일/BTC 7일)을 페이지 우상단 전역 토글(1D/1W/1M)로 일괄 전환. 정적 빌드 데이터에 기간별 시계열을 미리 포함해야 하므로 데이터 파일 용량과의 트레이드오프 검토 필요.
2. **앵커 순서 개인화** — 채권 중심 사용자는 채권·금리를 두 번째로 올리는 식의 그룹 순서 재배열(Settings 연동, 드래그 정렬). Hero와 앵커 바 자체는 고정 유지.
3. **지표 캘린더 알림** — 정적 환경 제약상 푸시는 불가하므로, 경제일정의 이벤트를 ICS 파일로 내려받아 사용자 캘린더에 등록하는 "캘린더에 추가" 액션이 현실적 대안.
4. **상관관계 미니 매트릭스** — USD/KRW·미10Y·WTI·KOSPI의 30일 상관계수 히트맵(회색 단계, R2 준수)을 변동성 섹션 옆에 추가하는 안. 정보 밀도 증가 대비 오독 위험(상관≠인과)이 있어 경고 문구를 반드시 병행해야 하는데, **[리뷰 반영]** 이 페이지는 더 이상 `accent.violet` 해석 레이어를 기본으로 쓰지 않으므로(§3-2 ①, §6) 병행 문구도 일반 캡션(`type.caption` + `text.secondary`, 굵기로 강조)으로 처리한다 — violet은 Dashboard·Financial Statements 전용으로 남겨둔다.
5. **서프라이즈 → 시장 반응 연결** — 경제지표 행에 "발표 직후 1시간 S&P 선물 반응(±%)" 열 추가. 서프라이즈의 무채색 원칙을 유지하면서 시장 방향 열에만 등락색을 쓰면 색 체계와 충돌하지 않는다.
6. **데이터 신선도 패널 연동 강화** — 사이드바 하단 "데이터 생성 시각" 클릭 시 열리는 신선도 패널(00 §6-5)에 이 페이지 27개 카드의 소스별 T·마지막 갱신·상태를 일람 표로 제공. STALE 다발 시 원인(빌드 실패 등) 진단 진입점이 된다.
7. **야간선물 세션 캘린더 예외 처리** — CME 휴장일·조기 폐장일에는 "다음 세션 18:00 개장" 카운트다운이 틀린 정보가 된다. 데이터 파일의 유효 세션 메타(00 §12)에 휴장 캘린더를 포함해, 카운트다운이 "다음 세션 07/14 (월) 18:00 개장"처럼 날짜까지 명시하도록 확장해야 한다.
