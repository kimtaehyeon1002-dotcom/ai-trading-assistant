# 06. Financial Statements

**AI Trading Assistant — Financial Statements 페이지 설계 v1.1**
근거 문서: 00. 디자인 시스템(Global Foundation v1.0). 본 문서는 00의 토큰명·수치를 변경 없이 인용한다.
기준 해상도: 1920×1080, Desktop First, 다크 테마 단일.
페이지 목표: **재무를 모르는 사용자가 색·형태만으로 3초 안에 '좋다/나쁘다'를 판독**하고, 아는 사용자는 3클릭 안에 원본 수치까지 도달한다(P5).

## 1. 전체 레이아웃 설명

### 1-1. 페이지 상태 2종

| 상태 | 트리거 | 구성 |

| **상태 A — 초기(검색 전)** | 페이지 최초 진입, 종목 미선택 | 페이지 타이틀 + 대형 종목 검색 필드 + 최근 조회 칩 + 관심종목 재무 스냅샷(Compact 카드 4장). 빈 화면 금지 원칙(§9-3)에 따라 검색 전에도 정보를 제공한다 |
| **상태 B — 분석 화면** | 검색 필드에서 종목 확정(자동완성 선택 또는 Enter) | 종목 컨텍스트 스트립 + 앵커 탭 + Hero(재무 종합 진단) + 5개 지표 그룹 섹션 + 원본 재무제표 아코디언 |

### 1-2. 상태 B의 수직 구조 (위→아래)

```
[1] 종목 컨텍스트 스트립   64px   sticky (헤더 64px 바로 아래 고정)
[2] 앵커 탭 바            44px   sticky (스트립과 한 몸으로 고정, 합계 108px)
[3] Hero — 재무 종합 진단  320px  12col
[4] 섹션: 성장성           260px  4col 카드 ×3
[5] 섹션: 수익성           260px  4col 카드 ×3
[6] 섹션: 안정성           260px  4col 카드 ×3
[7] 섹션: 현금흐름         260px(포커스 카드 승격 시 최대 476px)  6col+3col+3col(승격 시 12col 단독 행 + 6col×2 지원 행)
[8] 섹션: 밸류에이션        240px  4col 카드 ×3
[9] 원본 재무제표 아코디언   56px(접힘) / 최대 720px(펼침)  12col
```

- 섹션 간 수직 간격 32px, 섹션 헤더(`type.h2` 20px/600)와 첫 카드 사이 16px.
- 스크롤 0 뷰포트(1080px)에는 스트립+탭+Hero+성장성 카드 상단부가 들어온다. 즉 **첫 화면에서 결론(5축 진단)이 완결**된다(P5).
- **[4]~[8] 중 정확히 1개 그룹에서 §1-7의 "포커스 카드 승격"이 발생할 수 있다**(선정 대상이 없으면 승격은 생략된다). 승격이 일어난 그룹은 카드 3장이 "포커스 카드 1장(단독 행, 12col) + 나머지 2장(원래 스펙 그대로, 별도 지원 행)"으로 재편되며, 섹션 높이가 포커스 카드 높이(약 200~240px)와 내부 간격 16px만큼 늘어난다. 본 문서의 워크드 예시(삼성전자, 결산 2026 1Q)는 [7] 현금흐름 그룹에서 승격이 발생한 상태를 보여준다 — 승격 대상은 종목별 판정에 따라 달라지며 [4]~[6]이나 다른 위치에서 나타날 수도 있다(고정된 슬롯이 아니다).

### 1-4. 사이드바·헤더 특이사항 (재정의 없음, 00 §6·§7 그대로)

- 사이드바: **Financial Statements 항목 활성** — 좌측 3×20px `brand.primary` 인디케이터 바, 아이콘 `brand.primary`, 라벨 `text.primary` 600.
- 헤더 컨텍스트: 상태 A에서 `Financial Statements`(type.h3), 상태 B에서 브레드크럼 `Financial Statements / 삼성전자 005930`(구분자 "/", `text.tertiary` 12px).
- 글로벌 검색(Ctrl+K)에서 종목 선택 후 "Financial Statements" 페이지 그룹으로 진입하는 경로도 항상 열려 있다(§7-2).

### 1-5. 데이터 시간 축 — 3겹 as-of (N1 준수)

재무 데이터는 시세와 갱신 리듬이 다르므로 타임스탬프를 3겹으로 표기한다.

| 겹 | 예시 표기 | 위치 |
|---|---|---|
| 결산 기준 | `결산 2026 1Q (03/31)` | 종목 스트립 + 모든 카드 헤더 우측 |
| 공시 반영 | `공시 05/15 반영` | 종목 스트립 |
| 데이터 생성 | `데이터 생성 07/10 07:12 KST` | Hero 헤더 + 사이드바 하단 캡션(§6-5) |

신선도 판정 주기 T = **분기 공시 주기**(실시간 잣대 금지 — 00 §12). 차기 보고서 법정 제출 기한 경과 후 미반영이면 "지연" 배지(`accent.yellow` 아웃라인), 2분기 이상 미반영이면 STALE(값 `text.secondary` 강등 + "오래된 데이터" 배지). 시세가 아니므로 라이브 점·펄스·값 플래시는 이 페이지 전체에서 사용하지 않는다(§9-4 정신의 적용).

### 1-6. 판정(진단) 체계 — 이 페이지의 핵심 규약

모든 지표 카드는 **판정 3단계 + 삼중 부호화**를 따른다. (R4·§11-2 확장)

| 판정 | 색 | 배지 형태 | 의미 |
|---|---|---|---|
| **양호** | `status.success` #34D399 | 12% 채움 캡슐(radius 999px) + 색 도트 6px + 텍스트 "양호" | 기준선(업종 평균 등) 이상 |
| **중립** | `market.flat` #9CA3AF | 회색 아웃라인 캡슐 + 텍스트 "중립" | 기준선 부근·혼재 |
| **주의** | `status.warning` #FBBF24 | 12% 채움 캡슐 + 텍스트 "주의" | 기준선 미달·악화 추세 |

- **삼중 부호화**: ① 배지 색 ② 형태(게이지 마커가 기준선의 좌/우 어느 쪽인지) ③ 텍스트 라벨. 색을 못 봐도 위치와 글자로 동일 결론에 도달한다.
- **빨강 판정 없음**: 한국 모드에서 빨강 = 상승(`market.up` #F87171)이므로, "빨강 = 나쁨" 판정은 등락 반사신경과 정면 충돌한다. 최하 판정은 옐로 "주의"까지만 쓰고, 심각도는 텍스트("주의 — 3년 연속 악화")로 전달한다.
- **밸류에이션 그룹은 판정 미적용**: "싸다 = 좋다"는 단정이므로, 위치 서술("5년 밴드 하단 42%")만 회색 아웃라인 배지로 표기한다.
- 그룹 종합 판정: 소속 지표 전부 양호 → 양호 / 주의 1개 이상 → 주의 / 그 외 → 중립.
- 이 3단계 판정은 §1-7 "포커스 카드 승격" 자동 선정 로직의 유일한 입력값이기도 하다 — 판정 체계가 진단 표시뿐 아니라 레이아웃 강조 순서까지 결정한다.

### 1-7. 포커스 카드 승격 — 시각적 리듬 앵커 (신규)

15장의 지표 카드는 전부 §3-0의 동일한 6층 아나토미를 공유한다 — "한 번 배우면 15장을 같은 눈으로 읽는다"는 의도된 설계다(§5 참조). 그러나 색·크기·타이포가 전 카드에서 완전히 동일하면 어떤 지표도 두드러지지 않아, 긴 스크롤 동안 시각적 변주가 전혀 없는 재무제표 자동 출력물처럼 보일 위험이 있다. Bloomberg/TradingView급 제품은 핵심 지표 1~2개에 한해 카드 크기나 배지 강조를 달리해 "이게 지금 제일 볼 만하다"는 신호를 준다. 이 페이지는 그 신호를 **최대 1장의 카드**로, 그리고 **색이 아니라 크기로만** 만든다.

Hero(§3-1)가 페이지 전체의 결론 요약이라면, 포커스 카드는 지표 그룹 내부에서 스크롤 리듬을 깨는 국지적 앵커다 — 역할이 다르며 서로 대체하지 않는다.

#### 선정 규칙 (결정론적, 빌드 시점 계산 — 수동 큐레이션·AI 판단 없음)

1. 15장 중 **개별 카드 판정이 '주의'인 카드**가 있으면(→ 그 카드가 속한 그룹의 종합 판정도 자동으로 '주의'가 된다, §1-6), 그중 업종 평균 대비 편차(절댓값)가 가장 큰 카드를 승격 대상으로 선정한다.
2. '주의' 카드가 하나도 없으면(전 카드 양호 또는 중립), **판정이 '중립'인 카드** 중 업종 평균 대비 편차가 가장 큰 카드를 선정한다.
3. 전 카드가 '양호'로 동률이면(승격 대상 없음) 승격을 생략한다 — 앵커를 억지로 만들지 않는다. 이 경우 페이지는 §3-0의 표준 6층 카드 15장으로만 구성된다.
4. 밸류에이션 그룹(판정 미적용, §1-6)은 애초에 승격 대상에서 제외한다 — "싸다 = 좋다"는 해석과 뒤섞여 사실·해석의 경계를 흐릴 수 있으므로.
5. 승격 대상은 페이지 전체에서 최대 1장. 두 그룹에서 동시에 '주의'가 나와도 편차가 더 큰 1장만 승격한다(다중 승격은 리듬을 다시 무너뜨린다).

> **워크드 예시 적용**: 본 문서의 삼성전자(결산 2026 1Q) 예시는 성장성·수익성·안정성 3개 그룹이 전부 양호이고, 현금흐름 그룹의 잉여현금흐름(FCF)만 유일하게 중립(23년 적자 이력)이다. 규칙 2에 따라 FCF 카드가 승격 대상으로 선정된 상태를 이하 §2·§3에서 표준 사례로 보여준다.

#### 렌더링 스펙 — Standard-plus 카드 (00 §8-2 카드 등급 체계의 페이지 한정 확장)

| 항목 | 표준 카드(4col) | 포커스 카드(Standard-plus) |
|---|---|---|
| 그리드 스팬 | 4col (520px) | 12col (1616px), 그룹 헤더 직후 단독 행 |
| 높이 | 260px(대부분) / 240px(밸류, 승격 제외) | 200~240px |
| 6층 아나토미(§3-0) | 그대로 적용 | **동일하게 유지** — 레이아웃 문법을 깨지 않는다(§5 원칙 3) |
| 대표값 타이포 | `num-lg` 24px | `num-lg` 24px **그대로 유지** — `num-display`(32px)는 Hero 전용(00 §3-2)이므로 재사용하지 않는다 |
| 시각화 영역 | 높이 64~96px | 높이 120~140px, 기간 확장(예: 5년→10년) |
| 추가 레이어 | 없음 | 헤더 위 `type.overline` 11px `text.tertiary` 라벨 **"이번 분기 주목 지표"** + 본문 하단 컨텍스트 캡션 1문장(`text.secondary`, 왜 주목해야 하는지) |
| 색 | 판정 배지 색(§1-6) | **동일**, 새 색 미도입 — 강조는 크기·레이아웃으로만 만든다(색 예산 R1 불변) |
| 배치 | 그룹 3카드 행의 일부 | 그룹의 첫 행에 단독 배치, 나머지 2장은 원래 스펙 그대로 두 번째 행에 배치(압축·축소 없음) |

승격이 발생한 그룹은 §1-2의 표에 표기된 대로 섹션 높이가 늘어나며, 다른 그룹은 표준 높이를 그대로 유지한다. 이 페이지에서 "카드 크기"가 정보 신호로 쓰이는 지점은 이곳 한 곳뿐이다(색상 근거는 §6 참조).

### 3-0. 공통 지표 카드 아나토미 (00 §8-1 상속 + 페이지 확장)

모든 지표 카드는 아래 6층 구조를 공유한다. **한 번 학습하면 15장 전 카드가 같은 방식으로 읽힌다.**

| 층 | 요소 | 스펙 |

### 3-0-1. 포커스 카드(Standard-plus) 예외 — §1-7 참조

15장 중 최대 1장은 §1-7의 결정론적 규칙에 따라 12col Standard-plus로 승격될 수 있다. 승격 카드도 위 6층 아나토미를 그대로 유지하며(레이아웃 문법 통일 원칙은 깨지 않는다), 차이는 오직 다음 4가지뿐이다.

1. 그리드 폭: 4col → 12col(단독 행)
2. 시각화 영역 높이: 64~96px → 120~140px, 기간 확장(5년→10년 등)
3. 헤더 위 `type.overline` 라벨 "이번 분기 주목 지표" 추가
4. 본문 하단 컨텍스트 캡션 1문장 추가("왜 지금 이 지표를 봐야 하는지")

대표값 타이포(`num-lg` 24px)와 판정 배지 색은 표준 카드와 완전히 동일하게 유지해, 신규 색이나 신규 타입 스케일을 도입하지 않는다. "카드 문법 하나로 통일"(§5)이라는 원칙과 "리듬을 깨는 앵커가 최소 1곳 필요하다"는 요구를 동시에 만족시키는 지점이 바로 여기다.

- 스트립이 sticky이므로 현재가·등락을 놓치지 않고 재무를 본다(장중 컨텍스트 유지).

### 시나리오 3 — 저녁, 밸류에이션 심층 복기 (진입 후 +3클릭)

1. 상태 B에서 앵커 탭 **밸류에이션** — +1클릭. 스크롤 스파이로 탭 활성 전환(`motion.base` 200ms).

- 스트립의 [연간|분기] 토글로 전 카드가 12분기 창으로 일괄 전환 — 분기 흐름 복기.

### 시나리오 4 — 재무 초보자의 학습 동선 (클릭 최소, 호버 중심)

1. Hero에서 "현금흐름 ◦ 중립" 축 행 클릭 → 현금흐름 섹션으로 앵커 스크롤 — 1클릭.
2. 섹션 도착 시 FCF 카드가 이미 12col 포커스 카드로 확대되어 있어, 다른 카드보다 먼저 시선이 간다 — Hero에서 본 "중립" 판정이 스크롤 후에도 어떤 카드인지 즉시 식별된다(§1-7의 승격 효과).

4. 판정 배지·마커 위치·라벨이 항상 함께 있으므로, 용어를 몰라도 "0선 아래로 내려간 해가 있었다 → 중립"을 형태로 이해한다.

### 키보드 동선 (00 §11-3 상속)

- 첫 Tab: "본문으로 건너뛰기" → 페이지 검색 필드(상태 A) / 종목 스트립(상태 B).
- 앵커 탭 바에서 `←→` 이동 + Enter 스크롤, 카드 포커스 후 Enter = 확대 모달, Esc = 역순 해제.
- 등락·판정은 스크린리더에 "양호: 업종 평균 대비 1.8%포인트 높음"처럼 라벨 텍스트로 제공(기호·색 단독 노출 금지). 포커스 카드는 "이번 분기 주목 지표"라는 overline 텍스트가 스크린리더에도 그대로 읽혀, 시각적 확대와 동일한 정보를 비시각 사용자에게도 전달한다.

## 5. 디자인 의도

1. **판정 삼중 부호화가 이 페이지의 심장이다.** "3초 판독" 요구를 색에만 맡기면 색 절제 원칙(R1·R2)과 색맹 대응(§11-2)이 무너진다. 그래서 색(배지) + 형태(마커의 기준선 좌/우 위치) + 텍스트(양호/중립/주의)를 항상 동시 제공했다. 색을 지워도 결론이 남는 화면이 목표다.

2. **딱 1장만 승격해 스크롤 리듬을 깬다.** 15장이 전부 동일한 6층 카드면 "정보가 많다"가 아니라 재무제표 자동 출력물처럼 읽힐 위험이 있다. 그래서 §1-7의 결정론적 규칙(주의 판정 최우선, 없으면 중립 중 업종 대비 편차 최대)으로 딱 1장만 12col Standard-plus로 승격한다. 승격은 색이 아니라 **크기와 레이아웃**으로만 이뤄진다 — 색 예산(R1)을 넘기지 않으면서도 "이게 지금 제일 볼 만하다"는 신호를 스크롤 도중 최소 1곳에서 준다. 승격 대상이 없으면(전 카드 양호) 승격을 생략한다 — 억지 강조보다 정직한 반복이 낫다는 판단이다.

3. **표 대신 '카드 문법' 하나로 통일했다.** 15장의 지표 카드가 전부 6층 동일 구조(§3-0)라서, 사용자는 첫 카드 한 장만 학습하면 나머지를 자동으로 읽는다. Bloomberg급 밀도(카드당 대표값+기준선 2종+5년 추이)를 유지하면서 복잡해 보이지 않는 이유는 이 반복성이다 — "많아 보이되 복잡해 보이지 않기"(0-1)의 이 페이지식 해법. 단, §1-7에 따라 최대 1장은 예외적으로 승격되어 리듬을 깬다 — 반복성과 앵커는 양립한다. 예외가 15분의 1로 제한되어 있기 때문에 "학습된 카드 문법"이라는 전제 자체는 흔들리지 않는다.

4. **불릿 게이지와 레인지 바를 주력 시각화로 택했다.** 재무 지표의 본질은 "기준 대비 위치"다. 막대·파이보다 위치 판독이 빠르고, 업종 평균 ◇·자사 5y 평균 ▽·절대 경계 ¦를 한 트랙 위에 겹쳐도 밀도가 유지된다. 방향이 반대인 지표(부채비율)는 "← 낮을수록 양호" 캡션을 헤더에 상시 박아 형태 오독을 차단했다.

5. **밸류에이션에는 판정을 내리지 않는다.** PER이 낮은 것은 사실이지만 "좋다"는 해석은 투자 판단이다. 위치 서술(밴드 백분위)까지만 제공하고 회색 배지를 쓴 것은, 제품이 사실과 해석의 경계를 지킨다는 신뢰 장치다. 해석이 필요한 문장은 전부 violet 보더의 "진단 요약" 한 곳에 격리했다(`accent.violet` = 해석 레이어 전용, §2-3). 이 페이지는 00 §2-3-1이 지정한 violet 정식 사용 2개 페이지(Dashboard·Financial Statements) 중 하나다 — Macroeconomics·News·Asset 등 나머지 페이지가 동일 모티프를 일반 캡션으로 대체한 것과 대비되는 의도적 예외이며, 페이지 전체에서 단 1곳(Hero 진단 요약)에만 등장하기 때문에 "시스템이 짚어준 특별한 판단"이라는 신호력이 유지된다. 밸류에이션 그룹이 §1-7의 포커스 카드 승격 대상에서도 제외되는 것은 같은 논리의 연장이다 — 판정도 승격도 없는 그룹으로 남겨, 해석 개입 여지를 원천 차단한다.

6. **결론 우선 구조(P5).** Hero의 5축 진단은 아래 15장 카드의 목차이자 요약이다. 축 행 클릭 = 앵커 스크롤로, 요약과 상세가 같은 축 이름으로 1:1 연결된다 — "3초 파악 → 파고들기" 2단 구조의 문자 그대로의 구현. 포커스 카드 승격은 이 구조를 한 단계 더 보강한다 — Hero에서 "현금흐름 중립"을 본 사용자가 스크롤해 내려왔을 때, 어떤 카드가 그 판정의 근거인지 크기만으로 즉시 식별되게 한다.

7. **복잡한 표는 없애지 않고 격리했다.** 원본 재무제표는 최하단 접힘 아코디언과 확대 모달 안에만 존재한다. 초보자의 기본 경로에서는 보이지 않고, 파워유저는 +1클릭으로 도달한다.

8. **재무 데이터의 시간은 시세의 시간과 다르다(P3).** 결산 기준/공시 반영/데이터 생성의 3겹 as-of를 분리 표기하고, 라이브 점·펄스·플래시를 페이지 전체에서 금지했다. "2026 1Q 숫자"가 "지금 숫자"처럼 보이는 오독은 §9-4의 야간선물 사고와 같은 계열의 사고다.

## 6. 색상 및 여백 사용 이유

### 색 예산 회계 (R1 검증)

이 페이지의 뷰포트별 '의미 색'을 명시적으로 배정하고 상한 4종을 지킨다.

| 뷰포트 | 유채색 | 개수 |
|---|---|---|
| 스크롤 0 (스트립+Hero) | 등락 1색(스트립의 ▲빨강 또는 ▼파랑 중 하나) + 판정 그린 + 판정 옐로(있을 때) + violet(진단 요약) | ≤4 |
| 중간 섹션 (지표 카드들, 포커스 카드 포함) | 판정 그린 + 판정 옐로 + 차트 시리즈 2유채(#60A5FA·#A78BFA, 3번째 시리즈는 회색 #6B7280) | ≤4 |
| 상태 A (초기) | CTA 그린(검색 버튼 — 화면 유일의 Primary 채움, R3) + 자동완성 미리보기의 등락 1~2색 | ≤3 |

- 상태 B에는 Primary 채움 CTA가 0개다(종목 변경은 고스트 아이콘). R3는 "최대 1개" 규칙이므로 위반이 아니며, 판정 그린이 CTA로 오독될 여지를 없앤다.
- 등락 색은 **종목 스트립 한 곳으로 격리**했다. 카드 영역의 YoY 증감은 `text.primary` + 부호(＋/−, U+2212)로만 표기한다 — "매출 감소"가 파란 하락 시세로 오독되는 것을 막고, 색 예산도 지킨다(§3-3 "변화값만 market 색" 규칙의 의도적 페이지 예외로, 재무 증감은 시세 등락이 아니라는 판단).
- **포커스 카드(§1-7)는 이 색 예산 표에 새 항목을 추가하지 않는다.** 승격 카드도 판정 배지 색을 표준 카드와 동일하게 쓰므로, 뷰포트별 유채색 개수는 승격 여부와 무관하게 그대로 ≤4를 유지한다.

### 카드 크기의 리듬 — 색 대신 크기로 강조 (신규)

§1-7의 포커스 카드 승격은 이 페이지에서 유일하게 "카드 크기"가 정보 신호로 쓰이는 지점이다. 색으로 강조하면 색 예산(R1, 상한 4종)을 넘기거나 판정 색 체계(§1-6)와 충돌할 위험이 있으므로, 승격 카드는 배지 색을 표준 카드와 동일하게 유지한 채 **폭(4col→12col)과 시각화 높이(64~96px→120~140px)만으로** 주목도를 만든다. 이는 00 P1(Numbers First)의 이 페이지식 연장이다 — 강조가 필요할 때도 새 색을 추가하기보다 숫자·차트가 차지하는 면적을 키우는 쪽을 택한다. 15장 중 최대 1장이라는 상한(§1-7 규칙 5)을 지키기 때문에, 이 장치가 남용되어 페이지 전체가 다시 "전부 강조 = 전부 평범"으로 돌아가는 것도 구조적으로 막는다.

### 판정 색 선택의 근거

- **양호 = 그린(#34D399)이 가능한 이유**: 한국 모드(기본)에서 등락은 빨강/파랑이 전담하므로 그린은 등락과 충돌하지 않는다 — §2-5 근거 2("브랜드 색과 등락의 분리")가 만들어 준 여유를 판정 체계가 사용한다. `status.success`의 "긍정적 상태" 의미를 지표 판정으로 확장한 것이며, 채움 12% + 도트 + 텍스트 조합이라 채움 CTA(brand.primary)와 형태로도 구분된다.
- **나쁨 = 빨강을 쓰지 않는 이유**: 한국 사용자의 반사신경에서 빨강은 상승(좋음)이다. `status.danger`(#F87171)는 `market.up`과 동일 HEX라서 "주의 지표"가 "급등"으로 오독될 수 있다. 최하 판정을 옐로 "주의"로 캡핑하고 심각도는 텍스트로 전달하는 것이 이 제품의 색 체계에서 유일하게 안전한 선택이다.
- **violet의 단일 용도**: 진단 요약(해석 문장) 좌측 보더 3px, 페이지에서 단 1곳. 토큰 정의(해석 레이어 전용, 00 §2-3) 그대로이며, "이 문장은 데이터가 아니라 해석"이라는 시각적 면책 표시로 기능한다. 00 §2-3-1 개정에 따라 violet 정식 박스는 Dashboard와 Financial Statements 두 페이지에만 1차 배정되어 있다 — 이 페이지가 그 2곳 중 하나이므로 그대로 유지하되, 다른 여러 페이지에서 동일 형태가 반복되어 신호력이 옅어지던 기존 문제는 00 문서 개정(전용 페이지 지정 + 나머지 페이지 일반 캡션 대체)으로 해소되었다. 이 페이지 내부에서도 violet은 Hero 우측 웰 1곳에만 등장해, 페이지 내적으로도 남용을 피한다.
- 기준선(◇▽¦)은 전부 무채색 `#6B7280` — 기준선은 배경 정보이지 판정이 아니므로 색을 갖지 않는다(R2).

### 여백의 리듬

- **카드 내부 20px / 거터 24px / 섹션 간 32px**의 3단 리듬(4pt 스케일)이 "카드 안 < 카드 사이 < 그룹 사이"의 게슈탈트 근접성을 만든다. 6개 그룹의 경계를 보더나 배경색 없이 여백만으로 구분해, 유채색·장식 없이 구조가 읽힌다.
- 카드 내부의 **라벨↔값 4px / 값↔게이지 12px**은 "라벨과 값은 한 덩어리, 게이지는 그 증거"라는 위계를 만든다. 게이지 상하 12px 여백은 마커 라벨(num-xs)이 트랙과 겹치지 않는 최소 호흡이다.
- Hero 내부 3분할(500/560/384px)은 좌측에 가장 큰 숫자(P1), 중앙에 판정, 우측에 문장을 놓아 **숫자 → 판정 → 해석**의 읽기 순서를 폭 배분으로 강제한다.
- 불릿 트랙을 12px로 얇게 유지한 것은 채움 면적(색 면적)을 최소화하려는 색 절제의 연장이다 — 같은 정보를 더 적은 잉크로.
- 포커스 카드와 지원 행 사이의 16px 간격(§2-2)은 카드 내부 여백(20px)보다 좁고 섹션 간 간격(32px)보다 좁은 **중간 리듬**으로, "같은 그룹 안의 위계"(포커스 1장 + 지원 2장)를 여백 크기로도 한 번 더 확인시켜 준다.

## 7. 개선 포인트

1. **2종목 비교 모드**: 종목 스트립에 "+ 비교" 액션 추가 → 6col 좌우 분할로 동일 카드 문법을 나란히 렌더링. 카드 문법이 통일되어 있어 비교 모드 확장 비용이 낮다. 단, 포커스 카드(§1-7)는 두 종목의 승격 대상이 서로 다른 그룹일 수 있으므로, 비교 모드에서는 승격을 임시 비활성화하고 표준 6+3+3 레이아웃으로 통일해 좌우 대칭을 유지하는 규칙을 별도로 정의해야 한다.
2. **업종 분포 컨텍스트**: 불릿 게이지의 업종 평균 ◇ 마커를 업종 분포 스트립(사분위 음영, 무채색)으로 승격 — "평균보다 높다"에서 "업종 상위 18%"로 정밀화.
3. **진단 이력 타임라인**: 분기마다 5축 판정 스냅샷을 축적해 Hero에서 "직전 분기 대비 판정 변화(현금흐름 주의 → 중립)"를 표시. 판정의 방향성이 판정 자체보다 유용한 경우가 많다. 같은 이력 데이터를 §1-7 선정 로직에도 연결하면 "지난 분기에도 주목받았던 지표인지"를 포커스 카드 캡션에 병기할 수 있다.
4. **실적 캘린더 연동**: 다음 분기 보고서 제출 예정일 D-day 배지를 스트립에 표시하고, News 페이지의 실적 기사와 상호 링크.
5. **판정 기준 커스터마이즈**: Settings에 "보수적/표준/공격적" 판정 프리셋 — 임계값(예: 부채비율 양호 기준)을 사용자 성향에 맞게 조정. 기준 변경 시 모든 배지에 "사용자 기준" 캡션 병기로 투명성 유지. 프리셋이 바뀌면 판정 결과가 바뀌므로 §1-7의 포커스 카드 선정 결과도 함께 재계산되어야 한다.
6. **미국 종목 확장**: 10-K/10-Q 기반 동일 카드 문법 적용. 단위는 §3-3 규칙대로 상장 시장 기준($B/M, 소수 2자리)을 따르고 한 카드 안에서 혼용하지 않는다.
7. **정적 환경 제약(간단히)**: 서버가 없으므로 전 종목 온디맨드 분석은 불가하다. 관심종목 + 시총 상위 종목의 지표 JSON(값 + 결산·공시·생성 시각 메타 포함, §12)을 빌드 시 사전 생성하고, 검색 자동완성은 경량 인덱스 파일로 처리한다. 미생성 종목은 §3-8의 빈 상태("다음 빌드에 포함") 플로우로 수용한다.
8. **포커스 카드 선정 로직 고도화**: 현재는 "주의 최우선 → 중립 중 편차 최대" 2단 규칙(§1-7)이지만, 추후 사용자별 관심 지표 가중치(예: 배당 투자자는 배당수익률 그룹을 우선 — 단, 밸류에이션 그룹은 판정 미적용이므로 §1-7 규칙 4의 승격 제외 원칙과 별개로 "주목 배지"만 부여하는 식의 절충이 필요)를 반영한 개인화 승격 규칙으로 확장할 수 있다. 다만 규칙은 항상 결정론적이어야 하며(정적 사이트 제약, §12), 자동 선정 근거를 카드 내 캡션으로 투명하게 노출하는 원칙(§1-7 렌더링 스펙)은 어떤 확장에서도 유지한다.
