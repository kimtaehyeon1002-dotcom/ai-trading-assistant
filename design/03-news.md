Stock Hub 문서의 트리거 계약과 충돌.** v1.0의 §3-2는 펼침 영역의 관련 종목 칩을 "클릭 시 Stock 페이지 이동"으로 정의했으나, 05 §1-1 호출 지점 표는 이 칩("기사 본문·헤드라인에 붙은 티커 칩")을 Stock Hub(비모달 우측 슬라이드 패널)의 정식 트리거로 명시한다. 같은 상호작용에 대해 두 문서가 "오버레이로 열림"과 "페이지 이동"이라는 정반대 동작을 정의하고 있었다. v1.1은 §2-B, §3-2, 시나리오 2·3을 모두 "Stock Hub 오버레이 패널 오픈, 기사 본문·리스트 스크롤 유지, 페이지 이동 아님"으로 통일해 05와 동일한 계약을 따른다.
> - **③ [major] `accent.violet` "해석 레이어" 모티프가 5개 페이지·6곳 이상에서 반복돼 신호력이 희석됨.** 00 §2-3-1은 이 문제의 해소안으로 violet 좌측 바 박스를 **Dashboard Hero 요약과 Financial Statements 진단 요약, 단 두 곳**으로 좁히고, 그 외 페이지는 기본값을 좌측 바 없는 일반 캡션으로 되돌리기로 결정했으며 News를 그 "나머지 페이지"의 예시로 직접 지목한다. v1.1은 이 결정을 그대로 상속해 L3 중요도 게이지·엣지 바를 `text.tertiary → text.secondary → text.primary` 3단 명도 사다리로, AI 브리핑 카드 헤더·불릿을 아이콘 없는 일반 캡션으로 되돌렸다. §1-4, §3-2, §3-3, 와이어프레임, §5-2, §6-1에 반영.

## 1. 전체 레이아웃 설명

### 1-1. 페이지의 존재 이유와 핵심 컨셉

News 페이지는 "뉴스를 많이 보여주는 곳"이 아니라 **"시장에 실제 영향을 주는 뉴스만 통과시키는 관문"**이다. 이 편집 원칙은 문구로 주장하는 것이 아니라 UI 구조 자체로 체감되게 한다. 세 가지 장치로 구현한다.

1. **게재 카운터**: 리스트 카드 헤더에 "오늘 수집 412건 → 게재 37건"을 상시 표기한다. 사용자는 "걸러진 양"을 매일 눈으로 확인하며 필터의 존재를 신뢰하게 된다.
2. **전수(全數) 중요도 표기**: 게재된 모든 기사는 예외 없이 중요도 게이지와 영향시장 배지를 갖는다. "이 기사가 왜 실렸는가"에 대한 답이 행 안에 있다.
3. **당당한 빈 상태**: 기준을 통과한 뉴스가 없으면 빈 상태를 그대로 보여준다("최근 6시간, 게재 기준을 통과한 뉴스가 없습니다"). 뉴스가 없다는 사실 자체를 정보로 취급한다 — 채우기 위한 노이즈 기사를 절대 넣지 않는다.

### 1-2. 골격

- 사이드바(240px, 00 §6)·헤더(64px, 00 §7)는 전역 스펙 그대로. **이 페이지의 특이사항**:
  - 사이드바 활성 메뉴: **News** (좌측 3×20px `brand.primary` 인디케이터 바 + 아이콘 `brand.primary`).
  - 헤더 좌측 페이지 컨텍스트: `News` (`type.h3`). 브레드크럼 없음(1뎁스 페이지).
  - 헤더 글로벌 검색의 "뉴스" 결과 그룹은 이 페이지의 데이터와 동일 소스를 사용 — 검색 결과에서 Enter 시 해당 기사 행이 펼쳐진 상태로 이 페이지에 도착한다.
- 콘텐츠 영역: 최대폭 1600px, 12컬럼 / 거터 24px / 좌우 마진 32px, 헤더 아래 상단 여백 24px (00 §5-1).
- 수직 구성(위→아래):
  1. **탭 바** — 12col, 높이 48px. 탭 4개: `미국시장 / 한국시장 / 거시경제 / 속보`.
  2. **본문 2단 분할** — 좌측 뉴스 리스트 영역 8col(≈1,060px) + 우측 컨텍스트 레일 4col(≈516px). 거터 24px.

### 1-3. 탭 4개의 역할 정의

| 탭 | 콘텐츠 | 기본 정렬 | 갱신 기대 주기 T |
|---|---|---|---|
| 미국시장 | 간밤~현재 미국 증시·기업·섹터 뉴스 | 최신순 (중요도순 토글 가능) | 60분 |
| 한국시장 | 국내 증시·기업·수급·규제 뉴스 | 최신순 (중요도순 토글 가능) | 60분 |
| 거시경제 | 금리·물가·환율·정책·지정학 | 최신순 (중요도순 토글 가능) | 60분 |
| 속보 | 전 시장 통합, 발행 시각 타임라인 | **항상 최신순 (정렬 고정)** | 30분 |

- 탭 전환 시 좌측 리스트와 **우측 레일의 하위 카드(시장 스냅·캘린더)가 함께 전환**된다. 우측 최상단 "오늘의 뉴스 브리핑" 카드만 탭과 무관하게 고정 — 첫 화면 결론 요약(P5)을 담당한다.
- 탭 라벨의 신규 건수 배지: 마지막 열람 이후 추가된 기사 수를 `속보 ●3` 형태로 표시(점 6px + 숫자 11px). 신선도 FRESH일 때만 점이 `badge.live`, 그 외 `text.tertiary`.

### 1-4. 중요도 체계 — 3단계 결정

단계 수는 **3단계**로 확정한다. 5단계는 판정 근거가 흐려져 신뢰를 잃고, 2단계는 "게재됨 = 이미 중요"인 이 페이지에서 변별력이 없다. 3단계는 "지금 반드시 → 알아둘 것 → 맥락 참고"라는 투자자의 실제 행동 단위와 1:1 대응한다.

| 등급 | 명칭 | 판정 기준(예) | 하루 기대 건수 |
|---|---|---|---|
| L3 | **필수** (Critical) | 지수·환율·금리를 직접 움직였거나 움직일 이벤트 | 2~5건 |
| L2 | **주목** (High) | 시총 상위 종목·주요 섹터에 직접 영향 | 8~15건 |
| L1 | **참고** (Context) | 방향성 해석에 필요한 배경·후속 보도 | 10~20건 |

시각화 방식: **계단형 시그널 게이지 + 타이포 위계 + 좌측 엣지 바**의 3중 채널(색 단독 전달 금지 — 00 §11-2와 동일 사상). 세 채널 모두 색상(hue)이 아닌 **명도·형태·굵기**로만 구성된다 — 등락에 쓰이는 유채색(`market.*`)과 절대 섞이지 않으며, 아래에서 보듯 `accent.violet`도 쓰지 않는다.

- 게이지: 세로 막대 3개, 각 폭 3px·간격 2px, 높이 4/8/12px 계단형(총 13×12px). 채움 칸 수 = 등급.
  - L3 필수: 3칸 모두 `text.primary`(#F9FAFB) 채움.
  - L2 주목: 2칸 `text.secondary`(#9CA3AF) 채움, 3번째 칸 `border.subtle`.
  - L1 참고: 1칸 `text.tertiary`(#6B7280) 채움, 나머지 `border.subtle`.
  - 3단계가 `text.tertiary → text.secondary → text.primary`로 이어지는 하나의 **명도 사다리**를 이룬다. `accent.violet`은 이 게이지에 쓰지 않는다 — 00 §2-3-1 결정에 따라 "해석 레이어" 전용 박스(좌측 바 + overline + 문장)는 Dashboard Hero 요약과 Financial Statements 진단 요약, 단 두 곳에만 예약되어 있고, News를 포함한 나머지 페이지는 기본값인 무채색 위계를 상속한다(색을 걷어낸 배경은 §5-2 참고).
- 타이포 위계: L3 헤드라인 15px/600 `text.primary` · L2 14px/500 `text.primary` · L1 14px/400 `text.secondary`. 훑기만 해도 L3가 자연히 먼저 잡힌다.
- L3 한정 좌측 엣지 바: 행 좌측에 2×전체높이 `text.primary` 바 — 게이지 최상단 칸과 같은 명도를 재사용해 "사다리의 꼭대기"라는 신호를 형태로 한 번 더 반복한다. 스크롤 중 시야 주변부에서도 필수 기사를 놓치지 않게 하는 말초 시각 장치.
- 게이지 호버 시 Level 2 툴팁: "필수 — 지수·금리에 직접 영향" 등 판정 사유 1줄.

### 1-5. 스캔 최적화 — 밀도 2모드 + 행 펼침의 3단 깊이

"제목만 훑기 vs 펼쳐 읽기"를 **밀도 토글(리스트 전역) × 행 펼침(개별)**의 직교 구조로 해결한다.

| 깊이 | 트리거 | 행 높이 | 노출 정보 |
|---|---|---|---|
| ① 훑기 모드 | 밀도 토글 `☰` | 40px | 게이지 + 헤드라인 1줄 + 출처 + 시각 (한줄요약·배지 생략) |
| ② 기본 모드 | 밀도 토글 `≡` (기본값) | 72px | ① + 한줄요약 1줄 + 영향시장 배지 최대 2개 |
| ③ 펼침 | 행 클릭 / Enter | 72px + 확장 영역 ~148px | ② + 상세 요약 3줄 + 시장 반응(실측 등락) + 원문 링크 + 관련 종목 칩 |

- 밀도 토글 상태는 탭별이 아닌 페이지 전역으로 기억한다(사용자의 읽기 습관은 탭과 무관).
- 펼침은 동시 1행만 유지(아코디언). 다른 행 클릭 시 이전 행은 120ms로 접힘 — 스캔 리듬을 깨지 않는다.
- 키보드: `↑↓` 행 이동, `Enter` 펼침/접힘, `O` 원문 새 탭, `1~4` 탭 전환, `D` 밀도 토글.

### 1-6. 신선도 운영 (00 §9 적용)

- 리스트 카드 헤더에 **수집 시각**("수집 07/10 08:30 KST")을 생략 불가로 표기(N1). 개별 행의 시각은 **기사 발행 시각** — 두 시각의 역할이 다름을 툴팁으로 안내한다.
- 신선도 판정 주기 T: 일반 탭 60분, 속보 탭 30분. 경과 ≥3T이면 카드 헤더에 "오래된 데이터" 배지(`accent.yellow` 12% 채움) + 리스트 전체 시각 텍스트를 `text.secondary`로 강등(STALE).
- 뉴스는 시세가 아니므로 CLOSED-SNAPSHOT은 해당 없음. 단, 펼침 영역의 "시장 반응" 수치는 시세 데이터이므로 00 §9-4 세션 인식 규칙을 그대로 따른다(마감 후엔 "07/09 마감 기준" 라벨 병기, 라이브 점 금지).

- 헤더: `AI BRIEFING` `type.overline` 11px/600 자간 +0.4px, `text.tertiary` — 아이콘·좌측 바·색 강조 없는 표준 캡션 레이블. 우측 타임스탬프 "07/10 07:12 KST"도 동일한 `text.tertiary` 톤.
- 본문: 불릿 3개, 각 항목 좌측에 순번 숫자(①②③) `text.tertiary` 11px — 별도 배지·원형 배경 없이 텍스트만으로 순서를 표기한다. 본문 `type.body` 14px/400 `text.primary`, 핵심 수치(`▲+1.21%` 등)만 `type.num-sm` + `market.*` 색.
- **violet을 쓰지 않는 이유**: `accent.violet` "해석 레이어" 모티프(좌측 바 + overline + 문장)는 한때 Dashboard Hero 요약·Macro Hero 요약·News(이 카드 포함, L3 게이지·엣지 바까지)·FS 진단요약·Asset 인사이트까지 5개 페이지·6곳 이상에서 사실상 동일한 조형으로 반복되며 "시스템이 짚어준 특별한 판단"이라는 본래 신호력을 잃고 있었다(사내 리뷰 지적). 00 §2-3-1은 이 모티프의 "정식" 좌측 바 형태를 **Dashboard Hero 요약과 Financial Statements 진단 요약, 단 두 곳**으로 좁히고, 그 외 페이지의 기본값을 좌측 바 없는 **일반 캡션**(`type.caption` + `text.secondary`/`text.tertiary`)으로 되돌리기로 결정했다. News는 이 기본값을 그대로 상속한다 — 아이콘이나 원형 배지로 색을 우회해 넣지 않는다. 색을 다른 형태로 대체하는 것 자체가 "이 카드만의 정형화된 특수 위젯"을 새로 만드는 셈이라 원래 문제(반복으로 인한 신호 희석)를 다른 모습으로 재현하기 때문이다.
- 이 카드가 특별한 이유는 색이 아니라 **위치와 편집**에 있다: 탭을 전환해도 우측 레일 최상단에서 움직이지 않고(§1-3), 하루 중 가장 압축된 3줄만 싣는다. "항상 같은 자리에 있는 유일한 고정 카드"라는 사실 자체가 이미 충분한 신호이므로, 별도의 색 계약 없이도 우선순위가 흐려지지 않는다.
- 하단 요약행: "필수 2 · 주목 5 · 참고 7 (오늘)" — 12px `text.secondary`, 각 등급명 클릭 시 좌측 리스트가 해당 등급 필터로 전환(1클릭 드릴다운).
- 푸터: "전체 브리핑 보기 →" (Dashboard의 모닝리포트로 이동).
- 이 카드가 P5의 "첫 화면 결론 요약"을 담당한다: 3초 안에 "간밤에 무슨 일이, 오늘 뭘 주시해야 하는지"가 끝난다.

## 5. 디자인 의도

1. **편집 원칙을 '주장'이 아닌 '증거'로 제시한다.** "중요한 뉴스만 보여드립니다"라는 카피는 아무도 믿지 않는다. 대신 게재 카운터(412→37), 전수 중요도 게이지, 당당한 빈 상태라는 세 가지 구조적 증거를 배치했다. 특히 빈 상태를 회피하지 않는 결정이 핵심이다 — 노이즈로 채워진 뉴스 피드는 단 하루 만에 "여기도 똑같군"이라는 학습을 만들고, 그 순간 이 페이지의 존재 이유가 소멸한다.

2. **중요도를 '해석 레이어'로 분리하되, 색이 아니라 명도로 계약한다.** 중요도 판정은 시세(사실)가 아니라 편집(판단)이다 — 이 판단이 등락 색이나 경고 색을 입는 순간 사실과 판단의 경계가 무너진다는 원칙 자체는 그대로다. 다만 이 판단에 색을 입히는 방법은 v1.0에서 바뀌었다. 처음에는 `accent.violet`을 "해석 레이어"의 전용색으로 두고 L3 게이지·엣지 바·AI 브리핑을 모두 violet으로 묶었으나, 같은 좌측 바 + 회색 텍스트 조형이 Dashboard Hero 요약·Macro Hero 요약·News·FS 진단요약·Asset 인사이트까지 5개 페이지·6곳 이상에서 반복되며 "시스템이 짚어준 특별한 판단"이 아니라 어디서나 보이는 정형화된 위젯으로 소비되는 문제가 확인됐다(사내 리뷰 지적, 근거는 00 §2-3-1). 이에 따라 violet "해석 레이어" 박스는 Dashboard Hero 요약과 FS 진단 요약, 단 두 곳으로 용도를 좁혔다. News는 그 대신 **명도 사다리**로 같은 역할을 수행한다: 중요도 게이지·엣지 바는 `text.tertiary`(L1) → `text.secondary`(L2) → `text.primary`(L3)로 이어지는 3단 회색조로, AI 브리핑 카드는 좌측 바·아이콘 없는 일반 캡션(`text.tertiary` 오버라인 + 텍스트 순번)으로 내려온다(§3-3). 색 하나를 포기하는 대신 두 가지를 얻는다 — ① violet이 실제로 남은 두 페이지(Dashboard, FS)에서 신호력이 회복되고, ② News의 색 예산이 등락 2종 + brand 인디케이터 1종, 총 3종까지 줄어 00 §2-6(R1)·본 문서 §6-1의 4종 상한에 더 여유 있게 부합한다.

3. **스캔의 물리학: 위계는 읽기 전에 작동해야 한다.** L3 15px/600 → L1 14px/400 `text.secondary`의 타이포 경사와 계단형 게이지는 사용자가 글자를 읽기 전, 주변 시야 단계에서 이미 우선순위를 전달한다. 훑기 모드 40px 행은 1080px 뷰포트에서 약 18행이 노출되어 하루치 게재분 전체가 한 화면 반에 들어온다 — Bloomberg 급의 밀도를 유지하되, 복잡도는 회색 위계로 흡수한다(P1·P2의 동시 충족).

4. **실시간성을 연기하지 않는다.** 정적 배치 기반 서비스가 라이브 스트림처럼 꾸미는 것은 §9-4가 금지하는 "라이브처럼 보이는 스타일"의 뉴스판 재현이다. 속보 탭은 수집 주기·다음 갱신 예상·신규 항목 도트라는 정직한 어휘로 실시간성을 표현하고, 오래된 데이터는 공격적으로 강등한다. 신뢰는 빠름이 아니라 정확한 시간 표기에서 나온다.

5. **뉴스와 시세의 교차점을 '실측'에만 허용한다.** 펼침 영역의 "시장 반응"은 이 페이지에서 등락 색이 등장하는 유일한 지점이다. "이 뉴스는 호재"라는 예측 색칠은 하지 않는다 — 예측에 색을 입히기 시작하면 페이지 전체가 판단 오염된다. 실제로 움직인 자산의 실측치만 ▲▼와 함께 보여주고, 해석은 사용자와 브리핑 카드의 몫으로 남긴다.

6. **아코디언 단일 펼침과 날짜 페이지네이션.** 동시 다중 펼침을 금지한 것은 "지금 읽는 하나"에 집중시키고 스크롤 위치 감각을 보존하기 위함이다. 무한 스크롤 대신 날짜 단위 "더 보기"를 둔 것도 같은 이유 — 뉴스 복기는 "언제의 일인가"라는 시간 좌표가 생명이다.

| `market.up` #F87171 / `market.down` #60A5FA | 시장 반응·시장 스냅의 실측 등락 (▲▼ 병행) | 상시 (2종) |
| `brand.primary` #10B981 | 탭 활성 인디케이터·사이드바 활성 (소면적) | 상시 (1종) |
| `accent.yellow` #FBBF24 | "지연"·"오래된 데이터" 배지 | 이상 상태에만 |
| `badge.live` #34D399 | 신선도 점·신규 도트 (6~8px 점 한정) | FRESH일 때만 |

- `accent.violet`은 이 페이지에서 쓰지 않는다. 00 §2-3-1 결정에 따라 violet "해석 레이어" 박스는 Dashboard Hero 요약·FS 진단 요약 두 곳에만 예약되어 있고, News는 중요도 게이지·엣지 바·AI 브리핑 카드 모두 무채색 명도(§1-4, §3-3)로 대체했다(상세 근거 §5-2).
- 정상 상태의 화면은 등락 2색 + brand 인디케이터 = **3종**으로, R1 4종 상한보다 한 종 여유 있게 수렴한다 — violet을 걷어낸 만큼 색 예산에 여백이 생긴 것이지, 그 여백을 다시 다른 색으로 채우지 않는다(00 §2-3-1 "색 예산이 남았다는 이유만으로 violet 박스를 기본 채택하지 않는다"). yellow는 이상 상태의 침입자로만 등장하므로, 노랑이 보이는 것 자체가 "무언가 오래됐다"는 경보로 기능한다.
- 영향시장 배지·키워드 바·출처를 전부 회색 계열로 묶은 이유: 이들은 "분류"이지 "방향"이 아니다. 분류에 유채색을 쓰는 순간 "이 배지 색은 등락인가? 중요도인가?"라는 오독 비용이 발생한다(P2의 뉴스 카드 사례를 그대로 적용). 중요도 게이지·엣지 바의 명도 사다리도 같은 이유로 무채색을 유지한다 — 색이 아니라 밝기 차이로만 "판단"을 표현하면 등락색과 오독될 여지 자체가 없다.

### 6-2. 여백 논리 (4pt 스케일 준수)

| 위치 | 값 | 이유 |
|---|---|---|
| 탭 바 ↔ 본문 | 16px | 탭은 콘텐츠의 일부(필터)이므로 섹션 간격(32px)보다 가깝게 — "탭이 리스트를 지배한다"는 종속 관계 표현 |
| 리스트 ↔ 우측 레일 | 24px (거터) | 그리드 표준. 좌측은 '사실', 우측은 '컨텍스트'라는 병렬 관계 |
| 우측 레일 카드 간 | 24px | 카드 간 표준 간격 유지 — 레일이라고 압축하지 않는다 |
| 행 내부: 헤드라인 ↔ 요약 | 4px | 라벨↔값 표준. 한 기사임을 밀착으로 표현 |
| 행 ↔ 행 | 0 (구분선 1px `border.subtle`) | 리스트 밀도 확보. 분리는 여백이 아닌 보더가 담당(P4) |
| 펼침 웰 내부 패딩 | 16px | `surface.sunken` 웰의 Compact 패딩 규격 — "본문 속의 방"이라는 깊이 한 단계 |
| 우측 메타 열 고정폭 | 148px | 출처·시각의 세로 정렬 축 형성 — tabular 숫자와 함께 훑기의 수직 리듬을 만든다 |

- 행 높이 72px(기본)의 산출: 헤드라인 22px 행간 + 4px + 요약 20px 행간 + 상하 패딩 13px×2 = 72px. 40px(훑기)는 00의 밀집 Row 규격을 그대로 차용.
- 깊이 표현은 전면적으로 P4를 따른다: 리스트 카드(`surface.card` + `border.default`) 안의 펼침 웰(`surface.sunken`)은 "파인" 깊이, 툴팁(`surface.elevated` + 그림자)만 "뜬" 깊이. 같은 평면의 카드끼리 그림자 없음.

## 7. 개선 포인트

1. **키워드 구독**: 키워드 레이더의 키워드를 별(`accent.yellow`) 토글로 구독 → 해당 키워드 기사에 구독 마커 표시. 정적 환경에서는 localStorage로 저장하고, 구독은 알림이 아닌 "시각적 승격"임을 Settings에 고지.
2. **중복 기사 클러스터링**: 동일 이벤트를 다룬 복수 소스 기사를 "외 3건" 접힘으로 묶기 — 게재 카운터의 신뢰도를 높이고 리스트를 더 압축한다. 대표 기사 선정 기준(최초 보도 vs 최다 인용)은 빌드 파이프라인에서 결정.
3. **뉴스 ↔ 매매일지 연결**: 펼침 영역에 "이 뉴스를 일지에 인용" 고스트 버튼 → 매매일지 작성 화면에 기사 링크·헤드라인 프리필. 복기 동선(시나리오 3)의 클릭 수를 1회 단축한다.
4. **발표 전/후 상태 전환 자동화**: "오늘의 주요 일정" 카드가 클라이언트 시계 기준으로 발표 예정 시각 경과 시 "발표전" → "결과 대기" 캡슐로 자동 전환(§9-4 S4의 일정판 적용). 다음 빌드에서 실제치가 채워진다.
5. **영향시장 배지 → 크로스 페이지 필터**: 배지 클릭 시 해당 시장·섹터 필터로 Macroeconomics/Stock 페이지에 진입하는 딥링크. 현재는 리스트 내 필터까지만 지원.
6. **주말·휴장일 다이제스트 모드**: 휴장일에는 속보 탭 대신 "주간 필수 뉴스 요약" 뷰를 기본 노출 — 빈 타임라인보다 유용한 대체 콘텐츠.
7. 구현 제약 관련(간단히): 신선도 판정·상대시각·신규 도트는 모두 **빌드 시점에 데이터 파일에 기록된 수집 타임스탬프 + 배치 ID를 열람 시점 클라이언트 시계와 비교**해 산출한다(00 §12 전제와 동일). 기사별 시장 반응 수치는 수집 시점의 스냅샷이므로 반드시 세션 라벨("07/09 마감 기준")을 데이터에 동봉해야 하며, 라벨 없는 반응 수치는 렌더링하지 않는다(N1의 확장 적용).
